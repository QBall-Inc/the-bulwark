# P8.2 — Essential Agent Skills Repository Setup

## Overview

Fork independent, high-value skills from The Bulwark into a standalone GitHub repository ("Essential Agent Skills") for distribution to users who want quality-enforcement skills without the full Bulwark framework. Establish git subtree sync from Bulwark as source of truth, with a future path to GitHub Actions automation.

---

## 1. Problem Statement

The Bulwark project has produced several skills that are independently valuable:
- `anthropic-validator` — validates Claude Code assets against official standards
- `code-review` — structured code review with security, type safety, linting, and standards sections
- `bug-magnet-data` — curated edge case test data for boundary testing
- `test-audit` — T1-T4 test quality auditing with multi-stage pipeline

These skills are currently embedded in the Bulwark repo alongside Bulwark-specific infrastructure (governance protocol, pipeline orchestration, hooks). Users who want just the skills can't easily extract them.

## 4. Sync Strategy

### Phase 1: Git Subtree (Immediate)

Git subtree allows pushing a subdirectory from one repo into another while preserving history. Bulwark remains source of truth.

**Initial setup (one-time):**

```bash
# In Bulwark repo — add the standalone repo as a remote
git remote add essential-skills git@github.com:USER/essential-agent-skills.git

# Push each skill directory as a subtree
# Note: git subtree push works per-prefix, so we need a strategy
```

**Challenge:** Git subtree works with a single prefix (e.g., `skills/`), but we want to push only SOME skills, not all. Two approaches:

**Approach A: Push entire skills/ directory, curate in standalone repo**
```bash
git subtree push --prefix=skills essential-skills main
```
- Pushes ALL skills (including Bulwark-specific ones)
- Standalone repo uses `.gitignore` or post-push script to exclude Bulwark-specific skills
- Simple but leaks Bulwark-internal skills into standalone repo history

**Approach B: Script-based selective sync (Recommended)**
```bash
#!/bin/bash
# scripts/sync-essential-skills.sh
SKILLS=(
  anthropic-validator code-review bug-magnet-data test-audit
  test-classification mock-detection assertion-patterns component-patterns
  session-handoff subagent-prompting subagent-output-templating
)

DEST="/path/to/essential-agent-skills"

for skill in "${SKILLS[@]}"; do
  rsync -av --delete "skills/$skill/" "$DEST/skills/$skill/"
done

cd "$DEST"
git add -A
git status
echo "Review changes above, then: git commit -m 'Sync from Bulwark' && git push"
```
- Only syncs selected skills
- Clean — no Bulwark-internal skills leaked
- Manual trigger (user runs when ready)
- Preserves standalone repo's own README, LICENSE, etc.

**Recommendation: Approach B** — simpler, cleaner, more control. Git subtree's single-prefix limitation makes it awkward for selective sync.

### Phase 2: GitHub Actions (Future)

After the repo is established and manual sync is validated:

```yaml
# .github/workflows/sync-skills.yml (in Bulwark repo)
name: Sync Essential Skills
on:
  push:
    branches: [main]
    paths:
      - 'skills/anthropic-validator/**'
      - 'skills/code-review/**'
      - 'skills/bug-magnet-data/**'
      - 'skills/test-audit/**'
      - 'skills/test-classification/**'
      - 'skills/mock-detection/**'
      - 'skills/assertion-patterns/**'
      - 'skills/component-patterns/**'
      - 'skills/session-handoff/**'
      - 'skills/subagent-prompting/**'
      - 'skills/subagent-output-templating/**'

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Sync to essential-agent-skills
        run: |
          # Clone target repo
          git clone https://x-access-token:${{ secrets.SKILLS_REPO_TOKEN }}@github.com/USER/essential-agent-skills.git target

          # Sync each skill
          for skill in anthropic-validator code-review bug-magnet-data test-audit \
                       test-classification mock-detection assertion-patterns \
                       component-patterns session-handoff subagent-prompting \
                       subagent-output-templating; do
            rsync -av --delete "skills/$skill/" "target/skills/$skill/"
          done

          # Commit and push if changes exist
          cd target
          git config user.name "Bulwark Sync Bot"
          git config user.email "sync@bulwark.dev"
          git add -A
          if git diff --staged --quiet; then
            echo "No changes to sync"
          else
            git commit -m "Sync from Bulwark $(date +%Y-%m-%d)"
            git push
          fi
```

**Requirements for Phase 2:**
- GitHub Personal Access Token or Deploy Key with push access to standalone repo
- Store as repository secret `SKILLS_REPO_TOKEN`
- Triggered only when relevant skill paths change
