# P8.2 — Essential Agents & Skills Repository Setup

## Status: COMPLETED (Sessions 47-48)

## Overview

Fork independent, high-value skills and agents from The Bulwark into a standalone GitHub repository for distribution to users who want quality-enforcement skills without the full Bulwark framework. Uses rsync-based selective sync from Bulwark as source of truth, with a future path to GitHub Actions automation.

**Repo:** https://github.com/ashaykubal/essential-agents-skills

---

## 1. Problem Statement

The Bulwark project has produced several skills that are independently valuable:
- `anthropic-validator` — validates Claude Code assets against official standards
- `code-review` — structured code review with security, type safety, linting, and standards sections
- `bug-magnet-data` — curated edge case test data for boundary testing
- `test-audit` — T1-T4 test quality auditing with multi-stage pipeline
- `ez-statusline` — multi-line terminal status bar with context gauge

These skills are currently embedded in the Bulwark repo alongside Bulwark-specific infrastructure (governance protocol, pipeline orchestration, hooks). Users who want just the skills can't easily extract them.

## 4. Sync Strategy

### Phase 1: rsync-based Selective Sync (Implemented)

Git subtree was evaluated but rejected — it works with a single prefix but we need to sync only selected skills. rsync gives full control.

**Script:** `scripts/sync-essential-skills.sh`

Key behaviors:
- Syncs 11 named skills via rsync `--delete`
- Maps `bulwark-statusline` → `ez-statusline` with rebranding
- Bundles `statusline.sh` and config template inside the skill directory
- Copies `bulwark-standards-reviewer` → `standards-reviewer` agent
- Copies `statusline-setup` agent

### Phase 2: GitHub Actions (Future)

Automated sync on push to relevant skill paths. Requires `SKILLS_REPO_TOKEN` secret.

## 5. Key Decisions

### rsync over git subtree
Git subtree's single-prefix limitation makes it awkward for selective sync. rsync is simpler, cleaner, more controllable.

### Repo renamed to Essential Agents & Skills
User plans to add independent agents beyond Bulwark extracts. `agents/` directory included from the start.

### ez-statusline naming (Session 48)
`statusline` collides with Claude Code's built-in statusline command, causing duplicate menu entries. Renamed to `ez-statusline`.

### Self-contained skills (Session 48)
Skills must bundle their own scripts. `statusline.sh` moved from repo-root `scripts/` into `skills/ez-statusline/scripts/`. This principle applies to all skills — no root-level `scripts/` directory.

### Agent installation path
Agents install to `.claude/agents/` (project level), not a root `agents/` directory. The repo's `agents/` is for distribution; docs tell users where to place them.

### Executable bit fix (Session 48)
WSL/NTFS commits scripts as `644`. Used `git update-index --chmod=+x` to store `755` in git so clones on Linux/macOS work without manual `chmod`.

### Agent prompt includes exact JSON schema (Session 48)
`statusline-setup` agent prompt must include the exact `{"statusLine": {"type": "command", "command": "..."}}` structure, otherwise it produces invalid settings.json.
