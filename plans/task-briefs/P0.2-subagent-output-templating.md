## 1. Functional Requirements

### 1.1 Core Purpose

Provide a standardized template for sub-agent OUTPUT formatting. This skill complements P0.1 (subagent-prompting) which defines INPUT structure. Together they ensure:
- **P0.1**: How to prompt sub-agents (GOAL/CONSTRAINTS/CONTEXT/OUTPUT)
- **P0.2**: How sub-agents report results (log format, summaries, diagnostics)

This skill encodes Rules.md SA2-SA3 (Output to Logs, No Direct Context Pollution) into a reusable knowledge artifact.

### 1.2 Content Requirements

| Requirement | Description |
|-------------|-------------|
| FR-1 | Define YAML log file format with structured sections |
| FR-2 | Define task completion report template (WHY/WHAT/TRADE-OFFS/RISKS) |
| FR-3 | Define summary format for main thread (100-300 tokens, flexible) |
| FR-4 | Document diagnostic output format (consistent with P0.1) |
| FR-5 | Provide concrete examples of well-formed output |

### 1.3 User Stories

- As the **Orchestrator**, I need consistent log formats so I can parse sub-agent results
- As a **Pipeline Stage**, I need structured output so the next stage can consume my findings
- As a **Developer**, I need summary constraints so context isn't polluted with verbose output

### 3.2 Content Details

#### 3.2.1 YAML Log File Schema

Based on Loki Mode research and user requirements:

```yaml
# logs/{agent-name}-{YYYYMMDD-HHMMSS}.yaml

metadata:
  agent: bulwark-code-auditor
  timestamp: 2026-01-11T14:30:22Z
  model: sonnet
  task_id: "code-review-auth-module"
  duration_ms: 4520

goal: "Identify security vulnerabilities in authentication module"

completion:
  why:
    problem: "Authentication bypass vulnerability suspected"
    root_cause: "Token validation skips expiry check on refresh path"
    solution: "Added expiry validation to refresh token handler"

  what:
    - file: src/auth/token.ts
      lines: "45-52"
      change: "Added isExpired() check before token refresh"
    - file: src/auth/token.test.ts
      lines: "120-145"
      change: "Added test for expired refresh token rejection"

  trade_offs:
    gained:
      - "Security: Expired tokens now properly rejected"
      - "Compliance: Meets OWASP session management requirements"
    cost:
      - "Performance: Additional DB lookup on refresh (negligible)"

  risks:
    - risk: "Existing sessions with expired refresh tokens will fail"
      mitigation: "Grace period of 24h for migration"
      severity: medium

  next_steps:
    - "Monitor refresh failure rate for 24h"
    - "Remove grace period after migration window"

summary: |
  Found 1 critical vulnerability: refresh tokens not validated for expiry.
  Fixed by adding isExpired() check. Added regression test.
  Risk: existing sessions may fail (24h grace period applied).

diagnostics:
  model_requested: sonnet
  model_actual: sonnet
  context_type: forked
  parent_vars_accessible: false
  hooks_fired:
    - Stop
  execution_time_ms: 4520
  completion_status: success
```

#### 3.2.2 Summary Format Guidelines

| Aspect | Guideline |
|--------|-----------|
| Token Budget | 100-300 tokens (flexible based on complexity) |
| Must Include | Key findings, severity, action items |
| Must Exclude | Full reasoning, code snippets, verbose explanations |
| Format | Plain text or minimal markdown, no code blocks |
| Purpose | Enable main thread to decide next pipeline stage |

**Summary Template**:
```
Found [N] [severity] issue(s): [brief description].
[Action taken / recommended].
[Key risk or follow-up if any].
```

#### 3.2.3 Diagnostic Output Format

Consistent with P0.1 for behavioral testing:

```yaml
# logs/diagnostics/{agent-name}-{YYYYMMDD-HHMMSS}.yaml
skill: subagent-output-templating
timestamp: 2026-01-11T14:30:22Z
diagnostics:
  model_requested: sonnet
  model_actual: sonnet
  context_type: main  # or forked for agents
  parent_vars_accessible: true  # false for forked
  hooks_fired: []
  execution_time_ms: 1250
  completion_status: success
notes: "Output template applied successfully"
```
