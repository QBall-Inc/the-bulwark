### 3.2 Content Details

#### 3.2.1 4-Part Template (from Loki Mode research)

| Part | Purpose | Good Example | Bad Example |
|------|---------|--------------|-------------|
| GOAL | High-level objective | "Refactor authentication for maintainability" | "Refactor the auth file" |
| CONSTRAINTS | Explicit boundaries | "No new dependencies, maintain API compatibility" | (none specified) |
| CONTEXT | Required knowledge | "Files: auth.ts, Related: session.ts, Previous: failed validation" | "See the code" |
| OUTPUT | Concrete deliverables | "Log to logs/auditor-{timestamp}.md, summary 200 tokens max" | "Return results" |

#### 3.2.2 F# Pipeline Syntax

**Key Insight**: F# pipe syntax (`|>`) is a **conceptual notation** for planning/documentation, NOT directly executable Claude Code syntax.

```fsharp
// Conceptual notation (documentation)
CodeAuditor (security)
|> CodeAuditor (architecture)
|> TestAuditor (coverage)
|> (if findings > 0 then IssueDebugger else Done)
```

**Maps to actual execution**:
```markdown
1. Main thread invokes: Task(description="Security audit...", prompt="...")
2. Read log, extract findings
3. Main thread invokes: Task(description="Architecture audit...", prompt="...")
4. Read log, extract findings
5. Main thread invokes: Task(description="Test coverage audit...", prompt="...")
6. IF findings > 0: Main thread invokes IssueDebugger
7. ELSE: Done
```

#### 3.2.3 Model Selection Matrix

| Model | Primary Use | Cost | Examples |
|-------|-------------|------|----------|
| **Opus** | Planning/architecture ONLY | Highest | System design, security audits, novel problems |
| **Sonnet** | Development, complex analysis | Medium | Feature implementation, code review, integration tests |
| **Haiku** | Operations, simple tasks | Lowest | Unit tests, pattern matching, quick lookups |

**Critical Rule**: "Opus is ONLY for planning/architecture" - reserve expensive models for decisions, not execution.
