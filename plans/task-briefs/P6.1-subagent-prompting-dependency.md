# P6.1: Add subagent-prompting Dependency to Orchestration Skills

## Overview

Ensure all skills that spawn sub-agents include `subagent-prompting` in their `skills:` frontmatter array. This guarantees the 4-part prompt template and model selection rubric are available regardless of invocation path.

---

## Problem Statement

### Current State

The `suggest-pipeline.sh` PostToolUse hook injects instructions to load `subagent-prompting` when code is edited. This creates a dependency chain:

```
PostToolUse (Write|Edit) → suggest-pipeline.sh → instructs to load subagent-prompting
```

### The Gap

Skills invoked directly (e.g., `/bulwark-verify src/cli.ts`) bypass this chain because:
- No Write|Edit occurred
- suggest-pipeline.sh never fires
- subagent-prompting is not loaded

### Affected Skills

| Skill | Spawns Sub-Agents | Has subagent-prompting | Status |
|-------|-------------------|------------------------|--------|
| `bulwark-verify` | Yes (Sonnet) | No | Needs fix |
| `anthropic-validator` | Yes (claude-code-guide, bulwark-standards-reviewer) | No | Needs fix |
| `test-audit` | Yes (Haiku, Sonnet) | No | Needs fix |

### Working Examples

Custom agents already include this dependency:
- `bulwark-issue-analyzer` - has `skills: [subagent-prompting, ...]`
- `bulwark-fix-validator` - has `skills: [subagent-prompting, ...]`

---

## Solution

Add `subagent-prompting` to the `skills:` array of each affected skill.

### P2.3 bulwark-verify

```yaml
---
name: bulwark-verify
description: Generate runnable verification scripts...
user-invocable: true
skills:
  - subagent-prompting      # ADD
  - assertion-patterns
  - component-patterns
---
```

### P0.5 anthropic-validator

```yaml
---
name: anthropic-validator
description: Validates Claude Code assets...
user-invocable: true
skills:
  - subagent-prompting      # ADD
---
```

### P0.8 test-audit

```yaml

user-invocable: true

  - subagent-prompting      # ADD
  - test-classification
  - mock-detection

  - component-patterns
```
