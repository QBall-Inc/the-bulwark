# P5.9: Customizable Status Line - Task Brief

## Overview

Create a multi-line, customizable status line for Claude Code that displays session information with pastel colors and configurable segments.

## Visual Design

```
🤖 Opus 4.5  ▰▰▰▰▱▱▱▱▱▱ 42% (84K/200K)
Last file: tests/fixtures/test-audit/README.md
Git: the-bulwark/main (5 files pending commit)
```

## Config Schema

```yaml
# ~/.bulwark/statusline.yaml
preset: developer

lines:
  line1:
    segments: [model, context_gauge, token_counts]
  line2:
    segments: [last_file]
  line3:
    segments: [git_branch, pending_commits]

colors:
  gauge_low: "#AFFFAF"      # pastel green
  gauge_mid: "#FFF4B0"      # pastel yellow
  gauge_high: "#FF9A96"     # pastel coral
  gauge_empty: "#585858"    # dim gray
  model_opus: "#C4ADED"     # soft purple
  model_sonnet: "#ACD5F3"   # soft blue
  model_haiku: "#ACEFD6"    # soft teal
  label: "#8A8A8A"          # medium gray
  file_path: "#FFD7AF"      # pastel peach
  git_branch: "#87D7FF"     # pastel cyan

gauge:
  width: 10
  filled_char: "▰"
  empty_char: "▱"
  thresholds:
    low: 60
    high: 70
```

## Installation Flow

1. User runs `/bulwark:statusline-init`
2. Script copies default config to `~/.bulwark/statusline.yaml`
3. Script adds to `.claude/settings.json`:
   ```json
   {
     "statusLine": {
       "type": "command",
       "command": "${PLUGIN_ROOT}/scripts/statusline/statusline.sh",
       "padding": 0
     }
   }
   ```
4. User sees: "Status line installed! Customize at ~/.bulwark/statusline.yaml"

## Technical Notes

- **Multi-line confirmed** (Session 18, 2026-01-24): Script outputting multiple lines displays all lines in Claude Code status area
- Git pending count: `git status --porcelain | wc -l`
- Git repo/branch: `basename $(git rev-parse --show-toplevel)`/`git branch --show-current`
- YAML parsing: Use `yq` if available, fallback to `grep`/`sed`
- Script receives JSON via stdin from Claude Code
- Use RGB escape codes for exact hex colors (modern terminal support)
- **WSL note**: Ensure LF line endings on shell scripts (CRLF breaks shebang)
