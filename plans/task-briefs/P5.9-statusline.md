# P5.9: Customizable Status Line - Task Brief

## Overview

Create a multi-line, customizable status line for Claude Code that displays session information with pastel colors and configurable segments.

## Visual Design

```
🤖 Opus 4.5  ▰▰▰▰▱▱▱▱▱▱ 42% (84K/200K)
Last file: tests/fixtures/test-audit/README.md
Git: the-bulwark/main (5 files pending commit)
```

### Color Scheme

**Gauge thresholds:**
- <60%: pastel green
- 60-70%: pastel yellow
- >70%: pastel coral/red

**Model backgrounds (pastel):**
- Opus: soft purple (#183)
- Sonnet: soft blue (#153)
- Haiku: soft teal (#158)

**Segment labels:** dim gray
**Segment values:** pastel peach (files), pastel cyan (git)

---

## Presets

| Preset | Lines | Content |
|--------|-------|---------|
| `minimal` | 1 | Model + gauge + tokens |
| `developer` | 3 | Model + gauge + tokens, last file, git + pending |
| `cost` | 2 | Model + gauge + tokens + cost, duration |
| `custom` | 1-4 | User selects via interactive prompt |

---

## Available Segments

| Segment | Example | Required |
|---------|---------|----------|
| `model` | `🤖 Opus 4.5` | Yes |
| `context_gauge` | `▰▰▰▰▱▱▱▱▱▱ 42%` | Yes |
| `token_counts` | `(84K/200K)` | No |
| `cost` | `$0.42` | No |
| `duration` | `12m` | No |
| `last_file` | `Last file: path/to/file.ts` | No |
| `lines_changed` | `+42/-15` | No |
| `git_branch` | `Git: repo/branch` | No |
| `pending_commits` | `(5 files pending commit)` | No |

## Commands

| Command | Action |
|---------|--------|
| `/bulwark:statusline-init` | Install status line, copy default config |
| `/bulwark:statusline minimal` | Switch to minimal preset |
| `/bulwark:statusline developer` | Switch to developer preset |
| `/bulwark:statusline cost` | Switch to cost preset |
| `/bulwark:statusline custom` | Interactive segment selection |

## Config Schema

```yaml
# ~/.bulwark/statusline.yaml
preset: developer

lines:
  line1:
    segments: [model, context_gauge, token_counts]
  line2:
    segments: [last_file]
  line3:
    segments: [git_branch, pending_commits]

colors:
  gauge_low: 114      # pastel green
  gauge_mid: 222      # pastel yellow
  gauge_high: 210     # pastel coral
  gauge_empty: 240    # dim gray
  model_opus: 183     # soft purple
  model_sonnet: 153   # soft blue
  model_haiku: 158    # soft teal
  label: 245          # medium gray
  file_path: 223      # pastel peach
  git_branch: 117     # pastel cyan
  pending_count: 222  # pastel yellow

gauge:
  width: 10
  filled_char: "▰"
  empty_char: "▱"
  thresholds:
    low: 60
    high: 70
```

## Installation Flow

1. User runs `/bulwark:statusline-init`
2. Script copies default config to `~/.bulwark/statusline.yaml`
3. Script adds to `.claude/settings.json`:
   ```json
   {
     "statusLine": {
       "type": "command",
       "command": "${PLUGIN_ROOT}/scripts/statusline/statusline.sh",
       "padding": 0
     }
   }
   ```
4. User sees: "Status line installed! Customize at ~/.bulwark/statusline.yaml"

---

## Technical Notes

- Multi-line is achieved via sequential output (script outputs 3 lines)
- Git pending count: `git status --porcelain | wc -l`
- Git repo/branch: `basename $(git rev-parse --show-toplevel)`/`git branch --show-current`
- YAML parsing: Use `yq` if available, fallback to `grep`/`sed`
- Script receives JSON via stdin from Claude Code
