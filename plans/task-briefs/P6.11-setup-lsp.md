# P6.11 — Create setup-lsp Skill

## Overview

Create a skill that configures Language Server Protocol (LSP) integration for Claude Code projects. LSP replaces text-based grep/glob searches (~30-60s per query) with semantic code intelligence (~50ms per query), providing go-to-definition, find-references, type information, and real-time error detection.

This skill ships with the Bulwark plugin and integrates with `init` as an optional enhancement step.

## Implementation Method

Use `/create-skill` pipeline to generate this skill. This task brief serves as the input context for the create-skill interview stage.

---

## Functional Requirements

### FR1: Tech Stack Detection

Scan the project for language-specific manifest files to determine which language servers to install:

| Manifest File | Language | Server Plugin | Binary Install |
|--------------|----------|---------------|----------------|
| `package.json`, `tsconfig.json` | TypeScript/JavaScript | `typescript-lsp` | `npm i -g typescript-language-server typescript` |
| `pyproject.toml`, `requirements.txt`, `setup.py`, `Pipfile` | Python | `pyright-lsp` | `npm i -g pyright` |
| `go.mod` | Go | `gopls-lsp` | `go install golang.org/x/tools/gopls@latest` |
| `Cargo.toml` | Rust | `rust-analyzer-lsp` | `rustup component add rust-analyzer` |
| `pom.xml`, `build.gradle`, `build.gradle.kts` | Java | `jdtls-lsp` | `brew install jdtls` |
| `*.csproj`, `*.sln` | C# | `csharp-lsp` | `dotnet tool install -g csharp-ls` |
| `composer.json` | PHP | `php-lsp` | `npm i -g intelephense` |

For monorepos: scan subdirectories (1-2 levels deep) for additional manifest files. Present detected languages to user for confirmation before proceeding.

### FR2: Installation Pipeline

Sequential steps with restart/resume checkpoints:

1. **Enable LSP tool** — set `ENABLE_LSP_TOOL=1` in `~/.claude/settings.json` and shell profile
2. **Install binaries** — install language server binaries for detected languages
3. **Update marketplace** — `claude plugin marketplace update claude-plugins-official`
4. **Install plugins** — `claude plugin install <server>` for each detected language
5. **Enable plugins** — `claude plugin enable <server>` + add to `enabledPlugins` in settings
6. **CHECKPOINT: Restart required** — prompt user to restart Claude Code, resume session, and re-invoke skill to continue
7. **Verify initialization** — check `~/.claude/debug/latest` for `Total LSP servers loaded: N` where N > 0
8. **Report results** — summary of installed servers, verification status, any issues

### FR3: Restart/Resume Checkpointing

LSP servers initialize exclusively at startup. After plugin installation/configuration:
- Save progress state (which steps completed, which servers installed)
- Prompt user: "LSP plugins installed. Please restart Claude Code and resume session. Then run `/setup-lsp --verify` to complete setup."
- On `--verify` invocation: skip installation steps, run verification and troubleshooting

### FR4: Troubleshooting Diagnostics

When verification fails or on `--troubleshoot` invocation:

| Issue | Detection | Fix |
|-------|-----------|-----|
| Plugin installed but disabled | `claude plugin list` shows disabled | `claude plugin enable <name>` + restart |
| Missing binary in PATH | `which <binary>` fails | Reinstall binary, add to PATH |
| Stale marketplace catalog | "Plugin not found" error | `claude plugin marketplace update claude-plugins-official` |
| Startup race condition | "No server available" errors | Wait + restart |
| Zero servers loaded | Debug log shows `Total LSP servers loaded: 0` | Check all above, verify `ENABLE_LSP_TOOL=1` |

### FR5: Init Integration

When invoked from Bulwark init (P7.1):
- Run as optional step after core governance setup
- Ask user: "Would you like to set up LSP for faster code navigation? (Recommended)"
- If yes: run FR1-FR3 pipeline
- If no: skip, document how to run later (`/setup-lsp`)

## Technical Notes

### Known Issues

- `ENABLE_LSP_TOOL` flag is undocumented — community-discovered via GitHub #15619. May change or become default in future Claude Code versions.
- Java server (jdtls) takes ~8 seconds to initialize due to JVM warmup — this is normal.
- Other servers typically initialize in 0.5-0.6 seconds.
- First-run after plugin install may fail due to async loading race (#10997) — restart resolves.

### LSP Capabilities (for reference in skill description)

**Passive** (automatic after setup):
- Real-time error detection after edits
- Diagnostic pushing from language servers
- Self-correcting edits within single turns

**Active** (Claude invokes as needed):
- Go-to-definition
- Find all references/usages
- Type information (hover)
- Symbol listing and workspace-wide symbol search
- Implementation tracing for interfaces
- Call hierarchy (incoming/outgoing)
