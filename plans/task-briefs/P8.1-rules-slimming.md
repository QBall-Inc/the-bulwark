# P8.1 — Rules.md Slimming and Framework Independence

## Overview

Judicious trimming of Rules.md (~306 lines → ~185 lines) by removing code examples and moving Bulwark-specific sections (OR, SA, Grounding Clause) to CLAUDE.md project rules. Slim CLAUDE.md by removing duplication with Rules.md. Create portable copies for scaffold, init scripts, and a reasoning reference doc. All rule substance and critical context preserved.

---

## 1. Problem Statement

### 1.1 Token Cost

Rules.md is loaded every session (~306 lines). It contains:
- **Rules with elaboration** (the actual mandates + critical context) — ~60% of content
- **Code examples** (illustrative TypeScript/bash blocks) — ~15% of content
- **Bulwark-specific sections** (OR, SA, Grounding Clause) — ~25% of content

The rule elaboration is critical context that prevents misinterpretation and MUST be preserved. Code examples and Bulwark-specific sections can be removed/moved.

### 1.2 Framework Coupling

Current Rules.md contains both universal and Bulwark-specific rules:

| Rule Category | Universal? | Notes |
|---------------|-----------|-------|
| CS1-4 (Coding Standards) | Yes | Apply to any project |
| T1-4 (Testing) | Yes | Apply to any test suite |
| V1-4 (Verification) | Yes | Apply to any implementation |
| ID1-3 (Issue Debugging) | Yes | Apply to any bug fix |
| SC1-3 (Skill Compliance) | Yes | Apply to any Claude Code project with skills |
| TR1-3 (Task Rules) | **Partially** | Universal concept, Bulwark-specific paths |
| SR1-4 (Session Rules) | **Partially** | Universal concept, Bulwark-specific file refs |
| OR1-4 (Orchestrator) | **No** | Bulwark-specific (Implementer/Orchestrator modes) |
| SA1-6 (Sub-Agent) | **No** | Bulwark-specific (pipeline orchestration) |
| Grounding Clause | **No** | Bulwark-specific (Anthropic asset validation) |

### 1.3 CLAUDE.md Duplication

Current CLAUDE.md (~175 lines) duplicates significant Rules.md content:
- "Your Role" section (37 lines) duplicates OR1-4, OR2 model table, SA4 F# example
- "During Implementation" subsection duplicates V2, V4, SA2, TR1-3
- These duplications add token cost with no enforcement benefit

### 1.4 User Goal

Make Rules.md reusable across all future projects. The contractual obligation pattern (binding rules file → improved output quality) has proven effective and should be portable.

---

## 2. Proposal

### 2.1 Architecture: Two-Tier Rules (Option A — Chosen)

**Tier 1: Universal Rules (Rules.md)** — Framework-independent, portable across projects.
Contains: CS1-4, T1-4, V1-4, ID1-3, SC1-3, plus generalized versions of TR, SR.
Target: ~185 lines (from ~306). All rule substance preserved; only code examples removed.

**Tier 2: Project Rules (in CLAUDE.md)** — Project-specific extensions.
Contains: OR1-4, SA1-6, Task Conventions, Grounding Clause.

### 2.2 Key Design Decisions (Session 46)

1. **Rule elaboration is critical context, not fluff.** CS1's three bullets (Single Responsibility, Self-Contained, Independently Verifiable) are three distinct constraints. T3's test harness guidance is experience-born. These stay.
2. **Code examples can be trimmed.** T1/T2 TypeScript blocks, V2 bash block — the prose is sufficient.
3. **CLAUDE.md duplication is the real savings opportunity.** "Your Role" section drops from 37 lines to 2. "During Implementation" subsection removed entirely.
4. **Binding Contract table stays in CLAUDE.md.** CLAUDE.md takes precedence per model constitution; the reinforcement is necessary (proven by DEF-P4-005).
5. **OR1 reworked.** "Implementation is never delegated" was factually wrong (bulwark-implementer exists). Now: implementation is always Opus-class model.
6. **OR4 reworked.** Parallel sub-agents are now supported per pipeline-templates. Removes false "sequential only" claim.
7. **Model-agnostic language.** No version numbers (Opus, Sonnet, Haiku — not Opus 4.5/4.6).
8. **Portable copies for scaffold.** `lib/templates/rules.md` and `lib/templates/claudemd-injection.md` are the scaffold source files. The project root `Rules.md` and `CLAUDE.md` are for Bulwark development.

### 3.7 Init Scripts

**Script 1: `scripts/init-rules.sh`**

Copies `lib/templates/rules.md` (NOT the project root Rules.md) to a target project root.

```bash
#!/bin/bash
# Usage: init-rules.sh [target-dir]
# Copies universal Rules.md to target project root.
# If Rules.md already exists, creates backup.

TARGET="${1:-.}"
SOURCE_DIR="$(cd "$(dirname "$0")/.." && pwd)"

if [ -f "$TARGET/Rules.md" ]; then
  BACKUP="$TARGET/Rules.md.backup-$(date +%Y%m%d-%H%M%S)"
  cp "$TARGET/Rules.md" "$BACKUP"
  echo "Backed up existing Rules.md to $BACKUP"
fi

cp "$SOURCE_DIR/lib/templates/rules.md" "$TARGET/Rules.md"
echo "Rules.md installed at $TARGET/Rules.md"
```

**Script 2: `scripts/init-project-rules.sh`**

Injects content from `lib/templates/claudemd-injection.md` into target CLAUDE.md. Uses `--bulwark` flag to select Bulwark or blank project rules.

```bash
#!/bin/bash
# Usage: init-project-rules.sh [target-dir] [--bulwark]
# Injects Binding Contract + Mandatory Rules into CLAUDE.md.
# --bulwark: Include OR1-4, SA1-6 (Bulwark-specific rules)
# Without --bulwark: Inserts placeholder for custom project rules

TARGET="${1:-.}"
BULWARK=false
[[ "$*" == *"--bulwark"* ]] && BULWARK=true

SOURCE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
CLAUDE_MD="$TARGET/CLAUDE.md"

# Check if Mandatory Rules section already exists (idempotency)
if [ -f "$CLAUDE_MD" ] && grep -q "## Mandatory Rules" "$CLAUDE_MD"; then
  echo "Mandatory Rules section already exists in $CLAUDE_MD. Skipping."
  exit 0
fi

# Create CLAUDE.md if it doesn't exist
if [ ! -f "$CLAUDE_MD" ]; then
  echo "# Project Guide" > "$CLAUDE_MD"
  echo "" >> "$CLAUDE_MD"
fi

# Append injection content
cat "$SOURCE_DIR/lib/templates/claudemd-injection.md" >> "$CLAUDE_MD"

# Append project rules (bulwark or blank template)
if [ "$BULWARK" = true ]; then
  cat "$SOURCE_DIR/lib/templates/project-rules-bulwark.md" >> "$CLAUDE_MD"
else
  cat "$SOURCE_DIR/lib/templates/project-rules-template.md" >> "$CLAUDE_MD"
fi

echo "Mandatory Rules added to $CLAUDE_MD"
```

## 4. Portability Guide

For new projects, the user copies:

1. **Rules.md** — Drop in as-is. No edits needed.
2. **CLAUDE.md injection** — Run `init-project-rules.sh` to add Binding Contract + Mandatory Rules.
3. **governance-protocol** (optional) — Copy if using Bulwark hooks.

The universal Rules.md works standalone without governance-protocol or any Bulwark infrastructure.
