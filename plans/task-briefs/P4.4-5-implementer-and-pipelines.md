### Open Research Item: Hook Logging

**Question**: Can agent-scoped hooks access `$CLAUDE_PROJECT_DIR`?

The global hooks use `${CLAUDE_PLUGIN_ROOT}` (set by the plugin system) and `${CLAUDE_PROJECT_DIR}` (set by Claude Code). Agent-scoped hooks may or may not have the same environment variables.

**Validation approach**: The test scripts above use `$CLAUDE_PROJECT_DIR` - if the log entries appear, the variable works. If not, we need to find an alternative path reference (absolute path, relative path from agent working directory, etc.).

**Fallback**: If `$CLAUDE_PROJECT_DIR` is not available, use relative paths from the project root (the agent's working directory should be the project root).

### Agent Frontmatter

```yaml
---
name: bulwark-implementer
description: Code-writing agent with mandatory quality enforcement via agent-scoped hooks. Implements fixes and features following Bulwark standards.
model: opus
skills:
  - subagent-prompting
  - subagent-output-templating
  - component-patterns
tools:
  - Read
  - Grep
  - Glob
  - Write
  - Edit
  - Bash
hooks:
  PostToolUse:
    - matcher: "Edit|Write"
      hooks:
        - type: command
          command: "$CLAUDE_PROJECT_DIR/scripts/hooks/enforce-quality.sh"
          timeout: 60000
  Stop:
    - hooks:
        - type: command
          command: "$CLAUDE_PROJECT_DIR/scripts/agents/finalize-implementer.sh"
          timeout: 30000
---
```

**Note on hook command path**: Uses `$CLAUDE_PROJECT_DIR` based on research findings (Sec 7.1). If T-044 validation reveals this doesn't work in agent context, switch to relative path from project root.

**Note on enforce-quality.sh reuse**: The same `enforce-quality.sh` used by global PostToolUse hooks is reused for the agent. This means:
- Both agent-scoped AND global hooks fire on Write/Edit (parallel execution)
- Agent writes to `logs/` are skipped by enforce-quality.sh's path filter
- Code file writes trigger typecheck + lint + build
- The Phase 2 pipeline suggestion also fires (harmless for agent context)

### Agent Body Structure

The agent markdown body must include these sections in order:

1. **Pre-Flight Gate** (MANDATORY, first section after frontmatter)
   - Binding language per DEF-P4-005
   - Protocol step numbering
   - "REQUIRED" markers on critical steps

2. **Mission** (DO / DO NOT)
   - DO: Implement fixes, write tests, follow patterns, self-correct on hook failures
   - DO NOT: Skip quality checks, ignore hook feedback, write outside scope

3. **Invocation**
   - Task tool invocation pattern
   - Input structure for fixes vs features
   - Pipeline integration paths

4. **Protocol** (numbered steps)
   - Step 1: Parse input (fix mode vs feature mode)
   - Step 2: Read context (affected files, patterns, tests)
   - Step 3: Implement changes
   - Step 4: Write tests (using component-patterns)
   - Step 5: Handle hook feedback (self-correction loop, max 3 retries)
   - Step 6: Write outputs (report + diagnostics)
   - Step 7: Return summary

5. **Input Structure**
   - Fix mode: `debug_report_path`, `root_cause`, `affected_files`, `fix_approach`
   - Feature mode: `design_document`, `requirements`, `existing_patterns`

6. **Hook Failure Handling**
   - If enforce-quality.sh exits 2: read error output, fix violation, retry
   - Max 3 self-correction attempts per Write/Edit operation
   - After 3 failures: write partial report, return escalation to orchestrator
   - Document retry count in completion report

7. **Tool Usage Constraints**
   - Write: Source files, test files, logs/ (output reports)
   - Bash: Test runners, file execution, git read-only
   - Forbidden: git modifications, package installation, destructive commands

8. **Output Formats**
   - Implementation report: `logs/implementer-{id}-{YYYYMMDD-HHMMSS}.yaml`
   - Diagnostics: `logs/diagnostics/bulwark-implementer-{YYYYMMDD-HHMMSS}.yaml`
   - Summary: 100-300 tokens returned to orchestrator

9. **Completion Checklist**
   - All changes pass quality gates (typecheck, lint, build)
   - Tests written and passing
   - Hook retry count documented
   - Report written to correct path
   - Summary includes file paths for orchestrator

### finalize-implementer.sh

**Location**: `scripts/agents/finalize-implementer.sh`

**Purpose**: Stop hook script that runs when the implementer agent completes. Validates outputs and records metrics.

```bash
#!/bin/bash
# finalize-implementer.sh - Stop hook for bulwark-implementer agent
#
# Validates that expected outputs exist and records execution metrics.
# Called automatically when the implementer agent completes (Stop → SubagentStop).
#
# Input (stdin): JSON with agent_id, session_id, etc.
# Output: Validation results to hooks.log
#
# Exit codes:
#   0 = Finalization complete
#   (No blocking - Stop hooks are informational)

LOGS_DIR="${CLAUDE_PROJECT_DIR:-.}/logs"
mkdir -p "$LOGS_DIR"

INPUT=$(cat)
TIMESTAMP=$(date -Iseconds)
AGENT_ID=$(echo "$INPUT" | jq -r '.agent_id // "unknown"')

# Log finalization
echo "[${TIMESTAMP}] ImplementerStop: Agent ${AGENT_ID} completing" >> "$LOGS_DIR/hooks.log"

# Check for expected output files
REPORT_EXISTS="false"
DIAG_EXISTS="false"

# Check for implementer report (most recent)
LATEST_REPORT=$(ls -t "$LOGS_DIR"/implementer-*.yaml 2>/dev/null | head -1)
if [ -n "$LATEST_REPORT" ]; then
    REPORT_EXISTS="true"
    echo "[${TIMESTAMP}] ImplementerStop: Report found at ${LATEST_REPORT}" >> "$LOGS_DIR/hooks.log"
else
    echo "[${TIMESTAMP}] ImplementerStop: WARNING - No implementer report found" >> "$LOGS_DIR/hooks.log"
fi

# Check for diagnostics (most recent)
LATEST_DIAG=$(ls -t "$LOGS_DIR"/diagnostics/bulwark-implementer-*.yaml 2>/dev/null | head -1)
if [ -n "$LATEST_DIAG" ]; then
    DIAG_EXISTS="true"
    echo "[${TIMESTAMP}] ImplementerStop: Diagnostics found at ${LATEST_DIAG}" >> "$LOGS_DIR/hooks.log"
else
    echo "[${TIMESTAMP}] ImplementerStop: WARNING - No diagnostics found" >> "$LOGS_DIR/hooks.log"
fi

# Record to pipeline tracking
cat >> "$LOGS_DIR/pipeline-tracking.log" << EOF
[$TIMESTAMP] ImplementerStop
  agent_id: $AGENT_ID
  report_exists: $REPORT_EXISTS
  diagnostics_exists: $DIAG_EXISTS
  report_path: ${LATEST_REPORT:-none}
  diagnostics_path: ${LATEST_DIAG:-none}
EOF

exit 0
```
