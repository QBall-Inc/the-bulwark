#### FR2: Init Pipeline

Thin bash orchestrator that sets up governance files. Skills handle AI-guided configuration separately.

**Design principle:** Scripts handle file system operations. Skills handle AI-guided configuration. init.sh never invokes a skill.

```
scripts/init.sh (thin orchestrator)
  ├── MANDATORY: init-rules.sh → $TARGET/.claude/rules/rules.md
  └── PRINTED POST-INIT CHECKLIST:
       ├── "Run /setup-lsp for LSP support (recommended)"
       ├── "Run /bulwark-scaffold for Justfile + project tooling (recommended)"
       └── "Restart Claude Code session for hooks to take effect"
```

Changes to existing scripts:
- `scripts/init-rules.sh`: Update target path from `$TARGET/Rules.md` → `$TARGET/.claude/rules/rules.md`
- `scripts/init-project-rules.sh`: **Descoped from init pipeline.** CLAUDE.md injection is redundant with `.claude/rules/rules.md` auto-loading. Keep the file for reference but do not call from init.sh.

#### FR3: Cleanup Script (AC17)

Age-based cleanup for `logs/` and `tmp/` directories.

- Script: `scripts/hooks/cleanup-stale.sh`
- Threshold: T-10 days (`find -mtime +10`)
- Preserves `.gitkeep` files
- Runs on every SessionStart (no session type detection needed — T-10 threshold IS the guard)
- Hook config: Second entry in SessionStart hooks, timeout: 30

```bash
#!/usr/bin/env bash
# Age-based cleanup for logs/ and tmp/ directories
# Deletes files older than 10 days, preserves .gitkeep
find "${CLAUDE_PROJECT_DIR}/logs" "${CLAUDE_PROJECT_DIR}/tmp" \
  -type f -mtime +10 -not -name '.gitkeep' -delete 2>/dev/null
exit 0  # Never block session start
```
