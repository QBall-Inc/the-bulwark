## Technical Decisions

### Diagnostic Output Required for ALL Skills
- **Decision**: ALL skills must include diagnostic output, not just agents
- **Rationale**: Consistent observability enables better debugging and behavioral verification
- **Impact**: All future skill tasks must include diagnostic output section

### F# Pipeline Syntax is Conceptual Only
- **Decision**: F# pipe syntax (`|>`) is documentation notation, not executable
- **Rationale**: Claude Code doesn't execute F# - main thread makes sequential Task() calls
- **Impact**: Pipeline syntax documented as planning tool, with Task() mapping explained

### Flat Skills Directory Structure
- **Decision**: No atomic/composite nesting in skills/
- **Rationale**: Claude Code 2.1.x plugin structure requires flat skills/ at plugin root
- **Impact**: Removed skills/atomic/ and skills/composite/, all skills are direct children

### Skill Input vs Output Templating Split
- **Decision**: P0.1 covers INPUT structure (prompting), P0.2 covers OUTPUT structure (log format)
- **Rationale**: Separation of concerns - how to prompt vs how to report
- **Impact**: P0.2 will define log file format, summary requirements, diagnostic YAML structure
