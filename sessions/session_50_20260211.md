## Technical Decisions

### Self-contained scripts directory
- **Decision**: Scripts have own package.json, tsconfig.json, and jest config — independent from project root
- **Rationale**: Self-contained skills principle (Session 48 lesson). ts-morph is a heavy dependency that shouldn't leak into root package.json.
- **Impact**: Scripts typecheck independently. Root tsconfig doesn't need to include `skills/`.

### Jest + ts-jest for script tests
- **Decision**: Added jest/ts-jest to scripts devDependencies for testing AST scripts
- **Rationale**: Tests execute real scripts via `execFileSync` + tsx — need a test runner in the scripts directory
- **Impact**: `cd skills/test-audit/scripts && npx jest` runs all script tests. Root `just test` recipe deferred to P5.8.

### Pipeline execution on each script
- **Decision**: Full New Feature pipeline (Stages 4-5) on each script before moving to the next
- **Rationale**: User requested production-grade pipeline enforcement. Easier to fix issues per-script than batch.
- **Impact**: Higher token cost but better quality assurance. Both scripts approved with 0 critical/high findings.

### Pipeline deferrals (user-approved)
- **Decision**: Deferred Code Review pipeline for config files (package.json, tsconfig.json, Justfile) and Test Audit for T4 fixture
- **Rationale**: Config files contain no code logic. T4 fixture is the validation target itself (circular to audit).
- **Impact**: None — these are standard dependency declarations and thin Justfile wrappers.
