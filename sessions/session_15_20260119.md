## Technical Decisions

### Complexity-Driven Validation Depth
- **Decision**: Validation effort scales with fix complexity (from debug report)
- **Rationale**: Avoid "throwing kitchen sink at a typo" - token efficiency
- **Impact**: Low complexity = P1 only (3-8K tokens), High = full analysis (15-25K tokens)

### Call Site Analysis Approach
- **Decision**: Use Grep-based search initially, defer LSP to Future Enhancements
- **Rationale**: LSP is better but adds complexity; Grep works for initial implementation
- **Impact**: Section 12 documents LSP as P5 enhancement

### TestWriter Conditional
- **Decision**: Skip TestWriter if existing tests cover bug scenario
- **Rationale**: Avoid redundant test writing when coverage already exists
- **Impact**: Orchestrator evaluates debug report's validation_plan before invoking TestWriter

### Summary Report Paths
- **Decision**: Summary must include paths to full reports + explicit manual testing callout
- **Rationale**: Ensures orchestrator can surface details to user; no contradiction with summary-only output
- **Impact**: Summary format updated with report paths and manual testing fields
