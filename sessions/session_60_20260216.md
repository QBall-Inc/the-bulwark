## Technical Decisions

### Research-Evaluate-Deepen cycle (viewpoint agents)
- **Decision**: Add 5-step multi-pass reasoning to all viewpoint prompts (initial research → evaluate → identify gaps → deepen → reconcile)
- **Rationale**: Live testing showed single-pass agents produced shallow findings with superficial confidence labels
- **Impact**: Each viewpoint agent now does 2 research passes; word target bumped from 800-1200 to 1000-1500

### Propose-Challenge-Refine cycle (brainstorm roles)
- **Decision**: Add self-challenge reasoning to PM/Architect/Dev Lead; enable codebase exploration for Architect/Dev Lead
- **Rationale**: "Do not re-research" was too broad — prevented Architect/Dev Lead from validating proposals against actual codebase
- **Impact**: Architect/Dev Lead now explore codebase via Glob/Grep/Read; word target bumped from 1000-1500 to 1200-1800

### Critical Evaluation Gate (both skills)
- **Decision**: Classify user post-synthesis input before incorporating; spawn targeted follow-up agents for speculative/unvalidated claims
- **Rationale**: Live testing showed orchestrator blindly incorporated an invalid user suggestion that later required manual follow-up research to refute
- **Impact**: Research uses Direct Investigation + Contrarian (2 Sonnet) for speculative claims; Brainstorm uses Architect + Critic (2 Opus) for technical claims/architectural suggestions

### Highest-Risk Assumption focal point (Critic)
- **Decision**: Add explicit instruction for Critic to catalog all assumptions, rank by risk, and call out the single highest-risk assumption
- **Rationale**: Gives the synthesis and Critical Evaluation Gate a clear focal point for validation
- **Impact**: Critic output now includes explicit "Highest-Risk Assumption / If wrong / To validate" block

## Learnings

- **Architect over-engineers on blank-slate projects**: When there's no codebase to validate against, the Propose-Challenge-Refine cycle's "VALIDATE: explore the codebase" step finds nothing, leaving only self-challenge as a brake. The Critic is the real corrective force in greenfield scenarios. This is by design — the sequencing lets each role propose their ideal, then the Critic compresses to reality.
- **Critical Evaluation Gate works as intended**: The brainstorm synthesis correctly incorporated user preferences directly while the gate would have caught unvalidated technical claims. The Post-Synthesis Decisions section in the synthesis document shows proper attribution.
- **Checks and balances are the core value proposition**: The skill pipeline's value isn't preventing ambitious thinking — it's separating ideation from commitment. Architect thinks big (discovers patterns), Critic ensures none of it leaks into v1 without justification.
