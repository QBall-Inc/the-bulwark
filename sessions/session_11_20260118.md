## Technical Decisions

### Option B+ Scope
- **Decision**: Apply immediate fixes (always-flag, batching) now; defer AST scripts and advanced patterns to P5.10-12
- **Rationale**: Addresses most impactful gap (classification missing integration files without explicit mocks) with minimal work. AST scripts require TypeScript implementation, different skill set.
- **Impact**: P0.6-8 complete and functional for 50-100 file projects. Production hardening in P5.x.

### Parallel Batching Thresholds
- **Decision**: Classification batches at >20 files (batches of 20-25), Detection batches at >10 flagged files (batches of 10-15)
- **Rationale**: Based on context limits (~200K tokens) and clear-framework scale (87 files, 37K lines)
- **Impact**: Enables test-audit to handle real-world projects like clear-framework

### Always-Flag Rule for Integration/E2E
- **Decision**: Integration and e2e files always flagged for deep analysis regardless of mock indicators
- **Rationale**: T3+ violations use inline mock data (mockOrderData) without jest.mock() calls. Classification grep won't find these.
- **Impact**: Reduces false negatives for broken integration chain detection
