## Technical Decisions

### rsync Instead of git subtree
- **Decision**: Used rsync-based selective sync (Approach B) instead of git subtree
- **Rationale**: Git subtree works with a single prefix but we needed to sync only selected skills from `skills/`. rsync gives full control over which directories to copy.
- **Impact**: Sync is manual (run script), not automatic. GitHub Actions (Phase 2) will automate this.

### Repo Renamed to Essential Agents & Skills
- **Decision**: Renamed from "Essential Agent Skills" to "Essential Agents & Skills"
- **Rationale**: User plans to add independent agents (not just from Bulwark) in the future
- **Impact**: Repo name `essential-agents-skills`, `agents/` directory included from the start

### standards-reviewer Agent Included
- **Decision**: Moved `bulwark-standards-reviewer` agent as `standards-reviewer`
- **Rationale**: `anthropic-validator` skill spawns this agent. Without it, the skill is broken in standalone usage.
- **Impact**: Sync script updated to copy this agent. anthropic-validator README documents the requirement.

### Statusline Known Limitation Documented
- **Decision**: Documented that YAML config is not read by statusline.sh at runtime (dead code)
- **Rationale**: Fixing the script is out of scope for P8.2. Users should know preset switching has no effect.
- **Impact**: statusline README includes Known Limitations section

### Blanket Pipeline Deferral for Doc Writes
- **Decision**: User approved blanket deferral for all README/doc writes in standalone repo
- **Rationale**: 15+ doc file writes would each trigger pipeline hooks, consuming massive tokens for zero-risk documentation
- **Impact**: No pipeline logs for these writes. SA6 compliance maintained via explicit user approval.

## Learnings

- **gh CLI defaults to SSH remote**: Even when authenticated via HTTPS, `gh repo create --clone` sets up SSH remote. Had to `git remote set-url` to HTTPS for push to work.
- **WSL CRLF persists**: `sync-essential-skills.sh` got CRLF on first write. Fixed with `sed -i 's/\r$//'`. Always verify with `file` command.
- **Hook storm on doc-heavy tasks**: Writing 15+ documentation files triggers pipeline hooks on every single write. Blanket deferral is the practical approach — ask user once, apply to all.
- **Agent dependencies easy to miss**: `anthropic-validator` references `standards-reviewer` agent but the original plan didn't include it. User caught this during review. Always trace skill -> agent dependencies when extracting.
