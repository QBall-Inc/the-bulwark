## Learnings

- **Mandatory execution checklist position is critical**: Bottom-of-file checklists are ignored by Claude. Top-of-file with BINDING language is the only pattern that works. This extends DEF-P4-005 — Claude treats ALL instructions as advisory unless front-loaded with explicit binding language. Confirmed across create-skill (works) and plan-creation (failed without it, fixed in this session).
- **Memory was not the root cause**: TC3 Round 1 failure was initially attributed to project memory bias. Investigation revealed PM-Essentials has no MEMORY.md. The actual root cause was instruction positioning (checklist at bottom). Always investigate before assuming causation.
- **AT peer debate adds structural value over sequential**: Not just "more text" — AT produced new workpackages, changed technical decisions (fetch→node:https, rename→JSONL), and reconciled contradictions in real-time. Sequential mode can only document these as gaps for the orchestrator to resolve. This is the strongest empirical evidence for AT's value proposition.
- **Sonnet agents are viable for AT**: TC3 used Sonnet (user's choice) and quality was high. AT mode doesn't require Opus — offer both with cost/quality trade-off guidance.
