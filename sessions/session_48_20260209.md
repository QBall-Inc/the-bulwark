## Technical Decisions

### Option B for Bulwark path normalization
- **Decision**: Only move statusline.sh into its skill; leave scripts/hooks/ as shared infrastructure
- **Rationale**: Hook scripts are referenced by hooks.json globally, not owned by any one skill. bulwark-scaffold references them as templates to copy, not dependencies.
- **Impact**: Minimal churn now; full path canonicalization deferred to P7.1

### Executable bit via git update-index
- **Decision**: Used `git update-index --chmod=+x` instead of relying on filesystem permissions
- **Rationale**: WSL/NTFS reports all files as 777, so git defaults to 644. Native Linux/macOS clones would get non-executable scripts.
- **Impact**: All scripts now portable; no chmod needed after clone

### Agent prompt must include exact JSON schema
- **Decision**: Statusline-setup agent prompt includes literal JSON structure with "type": "command"
- **Rationale**: Without it, agent produced invalid settings.json missing required fields
- **Impact**: Applied to both repos; pattern to follow for any agent that writes structured config

## Learnings

- **WSL executable bit gotcha**: Always use `git update-index --chmod=+x` for shell scripts on WSL/NTFS. The filesystem can't represent the executable bit, so git defaults to 644 silently. This affects every clone on native Linux/macOS.
- **Agent prompts need explicit schemas**: When an agent writes structured config (JSON, YAML), the prompt must include the exact target structure. Agents will guess reasonable-but-wrong formats without it.
- **Self-contained skill principle**: Skills should bundle their own scripts and assets. No root-level directories that skills depend on. This applies universally, not just to the standalone repo.
- **Name collision with built-ins**: Always check if a skill name collides with Claude Code built-in commands before publishing. `statusline` was taken.
