## Technical Decisions

### Main Context Orchestration Pattern
- **Decision**: Skill will NOT use `context: fork` + `agent` in frontmatter
- **Rationale**: Sub-agents cannot spawn other sub-agents. Our workflow requires sequential invocation: `claude-code-guide` → `bulwark-standards-reviewer`
- **Impact**: Skill provides methodology/instructions; Claude orchestrates agents from main context

### Agent Frontmatter Structure
- **Decision**: Custom agent uses `model: sonnet` and `tools:` array, NOT `context: fork`
- **Rationale**: Agent is standalone, invoked by Task tool from main context
- **Impact**: Agent stored at `agents/bulwark-standards-reviewer.md`, copied to `.claude/agents/` for dogfooding

### Plugin Directory Structure
- **Decision**: All component directories (agents/, skills/, hooks/) at plugin root, NOT inside `.claude-plugin/`
- **Rationale**: Official Anthropic plugin structure requirement
- **Impact**: Only `plugin.json` goes inside `.claude-plugin/`

## Learnings

### Sub-Agent Spawning Constraint
- Sub-agents cannot spawn other sub-agents - this is a Claude Code architectural constraint
- For sequential agent workflows, use "Main Context Orchestration" pattern
- Skills with `context: fork` run entirely in isolated context, limiting orchestration capability

### Skills + Commands Merge
- Claude Code v2.1.3 merged skills and commands
- Skills can be invoked as `/skill-name arg1 arg2`
- `$ARGUMENTS`, `$1`, `$2` available for argument passing
- `user-invocable: false` hides from `/` menu
