## Technical Decisions

### PreToolUse → PostToolUse Design Change
- **Decision**: Changed from PreToolUse on Task to PostToolUse on Write|Edit
- **Rationale**: Research revealed PreToolUse cannot inject `systemMessage` to Claude; `additionalContext` only works on PostToolUse
- **Impact**: Hook now fires after code changes, suggests pipelines based on file type and change size

### "STOP - TASK INCOMPLETE" Wording
- **Decision**: Changed advisory wording to mandatory framing
- **Rationale**: "REQUIRED" and "MUST" weren't enough; Claude treated as optional due to task completion momentum
- **Impact**: Successfully triggers pipeline execution - E2E validated with modulo function

### File Type Thresholds
- **Decision**: Different thresholds for different file types (code <5, tests <10, docs <=10, config <3)
- **Rationale**: Different file types have different significance levels
- **Impact**: Small changes bypass silently, significant changes trigger pipeline

### Custom Agent Invocation
- **Decision**: Documented that custom agent `name` field is used as `subagent_type` in Task tool
- **Rationale**: Researched via claude-code-guide agent; confirmed official mechanism
- **Impact**: Clear guidance in subagent-prompting skill for custom vs built-in agents

## Learnings

### Hook Wording Matters
- "REQUIRED" and "MUST" are not enough to override task completion momentum
- "STOP - TASK INCOMPLETE" framing successfully interrupts the wrap-up phase
- Tying pipeline to task completeness is more effective than marking as mandatory

### PostToolUse additionalContext
- PostToolUse hooks can inject `additionalContext` that Claude considers
- This is advisory by nature, but strong wording can make it effective
- Full E2E flow: hook fires → additionalContext received → Claude acts on it

### SubagentStart/Stop Limitations
- `subagent_type` field shows "unknown" for custom agents in hook data
- Agent names are not passed through hook events
- Logging still works, just without agent name identification

### Pipeline Execution Pattern
- Claude correctly parallelized SecurityAuditor and TypeSafetyAuditor stages
- Conditional logic worked (QuickReviewer ran, not CodeReviewer, because no critical issues)
- Sub-agents successfully wrote YAML logs following output templating
