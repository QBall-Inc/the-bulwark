## Technical Decisions

### Build Step Added to Quality Gate
- **Decision**: Added `build` recipe to Justfile templates and enforce-quality.sh sequence
- **Rationale**: User identified that `npm test` triggers compilation, wasting tokens. Explicit build step ensures code compiles before testing.
- **Impact**: enforce-quality.sh now runs: typecheck → lint → build (NOT test)

### Hooks Default to Enabled
- **Decision**: `/bulwark:scaffold` generates hooks by default, `--no-hooks` to opt out
- **Rationale**: If someone uses Bulwark, they want enforcement. Opt-out is the exception.
- **Impact**: Changed from `--hooks` (opt-in) to `--no-hooks` (opt-out)

### Check Alias for CI Recipe
- **Decision**: Added `check: ci` alias to all Justfile templates
- **Rationale**: `ci` connotes automation, `check` is more intuitive for manual invocation
- **Impact**: Developers can use `just check` or `just ci` interchangeably

### Framework Detection Deferred to P6.2
- **Decision**: Framework-specific templates (Astro, Next.js, SvelteKit, etc.) deferred to optional enhancement
- **Rationale**: Node template covers 80% of cases. Framework detection adds complexity without blocking core value.
- **Impact**: P3 stays focused on enforcement infrastructure; P6.2 handles polish
