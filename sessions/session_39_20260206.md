## Learnings

- **Sub-agent context isolation**: Sub-agents spawned via Task tool run in their own context window. They do NOT inherit the parent's conversation history. Bias transmission is only through the orchestrator's prompt wording, not context leakage.
- **context: fork breaks pipeline patterns**: If a skill uses `context: fork`, it runs as a sub-agent. Sub-agents can't spawn other sub-agents. So skills that orchestrate pipelines (like code-review spawning 4 agents) CANNOT use `context: fork`.
- **Security bias in single-agent reviews**: Anthropic models over-index on security findings when reviewing all aspects at once. Pipeline's separation of concerns (one agent per section) prevents this.
- **PostToolUse can't truly block**: Exit 2 on PostToolUse shows stderr to Claude but can't undo the write. Use `decision: "block"` in JSON output to prompt the agent to self-correct. This is the correct pattern for quality enforcement.
- **Stop hooks auto-convert**: Declaring `Stop` in agent frontmatter auto-converts to `SubagentStop` at runtime. This is the agent finalization mechanism.
