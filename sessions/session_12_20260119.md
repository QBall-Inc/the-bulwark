## Technical Decisions

### P1.1 (bulwark-test-auditor) Skipped
- **Decision**: Skip P1.1, proceed directly to P1.2
- **Rationale**: Sub-agents cannot spawn other sub-agents. Test-audit requires 3-stage orchestration (classification → detection → synthesis). Main Context Orchestration pattern works - orchestrator loads test-audit skill and spawns sub-agents directly. Wrapper agent would be redundant.
- **Impact**: P1 reduced from 3 tasks to 2. Test-audit invocation remains skill-based.

### P1.2 as Wrapper Agent
- **Decision**: P1.2 (bulwark-issue-analyzer) IS a valid wrapper agent
- **Rationale**: Single-stage operation (no internal sub-agent spawning), benefits from isolated context (bias avoidance), encapsulates behavior. Loads issue-debugging + subagent-output-templating skills.
- **Impact**: Clean interface for Fix Validation pipeline Stage 1.

### Tool Constraints for P1.2
- **Decision**: Use instruction-based constraints, not tool deny-listing
- **Rationale**: Claude Code doesn't support path-based Write restrictions. Agent needs Write for logs, Bash for investigation.
- **Impact**: Agent instructions specify: Write only to `logs/`, Bash no destructive commands (rm, git push, etc.)

### Timestamp Format Standardization
- **Decision**: Standardize on `{YYYYMMDD-HHMMSS}` for file paths, `{ISO-8601}` for YAML fields
- **Rationale**: Current ambiguity (`{timestamp}`, `{ts}`, `{YYYYMMDD-HHMMSS}`) caused test-audit pipeline failures during dogfooding
- **Impact**: 7 files need updates before P1.2 implementation

## Learnings

- **Main Context Orchestration vs Wrapper Agents**: Multi-stage pipelines (like test-audit) must use Main Context Orchestration because sub-agents can't spawn sub-agents. Single-stage operations (like issue-analyzer) can use wrapper agents for encapsulation and isolation.
- **Dogfooding catches issues**: The timestamp ambiguity was discovered through actual use of test-audit pipeline, not theoretical review.
