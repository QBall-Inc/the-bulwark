## Technical Decisions

### OR1 removed — model selection via table only
- **Decision**: Removed "Implementation work is always performed by an Opus-class model" rule. Added "implementation" to Opus row in OR1 table instead.
- **Rationale**: The orchestrator is whatever model the user runs. Prescribing Opus is an artificial constraint. The OR1 table provides guidance for sub-agent model selection including implementation delegation.
- **Impact**: OR numbering shifted: old OR2→OR1, OR3→OR2, OR4→OR3. All references updated in both Rules.md and template claude.md.

### TR2 (Plan Structure) removed
- **Decision**: Removed "Plans must include: Functional Requirements, Technical Requirements..." rule.
- **Rationale**: plan-creation skill enforces structure. The rule predated the skill and is now redundant.
- **Impact**: TR numbering shifted: old TR4→TR2, old TR5→TR3.

### Modes of Operation lives in CLAUDE.md, not Rules.md
- **Decision**: Removed MA section from Rules.md. Template claude.md is the sole home.
- **Rationale**: CLAUDE.md is the operational document. Modes are operational, not governance rules.
- **Impact**: Grounding Clause (GC) remains in Rules.md as it's a validation rule.

### SA1/SA2 reference skills instead of inlining templates
- **Decision**: SA1 references subagent-prompting skill. SA2 references subagent-output-templating skill.
- **Rationale**: Skills ship with the plugin. Single source of truth. Saves ~8 lines.
- **Impact**: Rules.md no longer contains the 4-part template inline or the closed-loop language.

### Task Rules scoped to in-session tasklist
- **Decision**: TR rules govern the in-session TaskCreate/TaskUpdate system, not project plans.
- **Rationale**: plan-creation skill outputs plans/{slug}/plan_v{N}.md with phases/workpackages. In-session tasks are sub-tasks of those workpackages. TR1 links them: "Every task in tasklist should be traceable to an item in an implementation plan."
- **Impact**: TR1 reworded to make this explicit.

### Line separators removed from Rules.md
- **Decision**: Removed all `---` separators between sections.
- **Rationale**: No decision-making impact for models (they parse by headings). Saves ~10 lines of tokens.
