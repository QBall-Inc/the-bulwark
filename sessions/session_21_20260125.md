## Technical Decisions

### Task Persistence via claude-project Script
- **Decision**: Created launcher script that auto-detects project and sets `CLAUDE_CODE_TASK_LIST_ID`
- **Rationale**: direnv doesn't work with Claude Code's shell; env var must be set before launch
- **Impact**: Tasks now persist to `~/.claude/tasks/{project}/` across sessions

### Unbiased Test Fixtures
- **Decision**: Replaced template-matching fixtures with genuinely different implementations
- **Rationale**: Original fixtures matched skill template patterns, biasing test results
- **Impact**: Tests now validate skill analyzes actual code, not just pattern matching

### Statusline Bug Fix
- **Decision**: Added `|| true` to xargs pipeline in statusline.sh
- **Rationale**: xargs returns 123 on subcommand failure; with `set -e` this killed script silently
- **Impact**: Statusline now robust against file system edge cases

- Files created via Write tool on WSL can have CRLF endings
- Breaks bash scripts with cryptic "required file not found" error
- Fix: `sed -i 's/\r$//' filename`

- Tasks stored in `~/.claude/tasks/{id}/`
