## Technical Decisions

### TestAudit After TestWriter
- **Decision**: Add TestAudit stage (mock-detection) after TestWriter in Fix Validation pipeline
- **Rationale**: Ensures generated tests don't have T1-T4 violations before proceeding
- **Impact**: Pipeline now: `TestWriter |> TestAudit` when tests are written

### /fix-bug Skill Creation
- **Decision**: Create user-invocable skill to trigger Fix Validation pipeline deterministically
- **Rationale**: Conversational prompts caused Claude to skip pipeline stages; explicit invocation needed
- **Impact**: Users can run `/fix-bug path "description"` for guaranteed pipeline execution

### Bias-Free Test Fixtures
- **Decision**: Remove all hint comments and debug reports from fixtures; represent pre-fix state
- **Rationale**: User feedback that comments like "// FIX APPLIED" biased agent investigation
- **Impact**: Full pipeline tests now exercise IssueAnalyzer → FixWriter → TestWriter → FixValidator

### Relative Paths in Skills
- **Decision**: Use relative paths in skill references with note about `skills/` vs `.claude/skills/`
- **Rationale**: Same skill content works in both locations
- **Impact**: Related Resources table uses relative paths

## Learnings

### Pipeline Invocation Patterns
- Conversational prompts like "investigate and fix" may skip pipeline stages
- Explicit pipeline name or `/fix-bug` skill ensures deterministic execution
- Agent descriptions in frontmatter help Claude choose correct agent, but not guaranteed

### Test Fixture Design
- Bias-free fixtures (no hint comments) test real agent capability
- Pre-fix state fixtures exercise full pipeline, not just validator
- Debug reports should be generated by pipeline, not pre-created
