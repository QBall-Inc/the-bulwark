## Technical Decisions

### Judicious Trimming (Not Aggressive)
- **Decision**: Keep rule elaboration, remove only code examples and Bulwark-specific sections
- **Rationale**: Rule elaboration is critical context (CS1's three bullets are three distinct constraints, T3's test harness guidance is experience-born). One-line-per-rule format proposed initially would lose nuance.
- **Impact**: Rules.md 35% reduction instead of 75%, but all rule substance preserved

### OR1 Reworked: "Opus-class" Not "Never Delegated"
- **Decision**: OR1 now says implementation is always Opus-class model (primary or sub-agent like bulwark-implementer)
- **Rationale**: "Implementation is never delegated" was factually wrong — bulwark-implementer exists and is Opus
- **Impact**: Accurately reflects dual implementation paths

### OR4 Updated: Parallel Execution Supported
- **Decision**: OR4 now says "sequential by default; parallel supported where documented in pipeline-templates"
- **Rationale**: Pipeline-templates already defines parallel patterns (Code Review 4 parallel agents)
- **Impact**: Removes false "sequential only" claim

### Model-Agnostic Language
- **Decision**: No version numbers in OR rules (Opus/Sonnet/Haiku, not Opus 4.5/4.6)
- **Rationale**: Models change. Users may use different default models.
- **Impact**: Rules stay valid across model generations

### Portable Copies for Scaffold
- **Decision**: Project root Rules.md/CLAUDE.md are for Bulwark dev; lib/templates/ copies are for scaffold
- **Rationale**: Scaffold needs its own source files that don't include Bulwark-specific CLAUDE.md sections
- **Impact**: Two copies of Rules.md to keep in sync (documented in task brief Section 8.1)

### TR4-TR5: Task Identification Rules
- **Decision**: Added TR4 (unique T-NNN identifiers, dependency verification) and TR5 (durable descriptions)
- **Rationale**: Tasklist persists across sessions via CLAUDE_CODE_TASK_LIST_ID; needs stable references and self-contained descriptions
- **Impact**: All future task creation must include T-NNN identifier and set dependencies

## Learnings

- **Rule elaboration is context, not fluff**: The initial proposal to slim Rules.md to ~80 lines would have lost critical constraints. CS1's three bullets (Single Responsibility, Self-Contained, Independently Verifiable) are three distinct rules living under one ID. Judicious trimming preserves substance.
- **CLAUDE.md duplication was the real savings**: The 37-line "Your Role" section and 8-line "During Implementation" were pure duplication of Rules.md content. Removing these and adding project rules resulted in zero-duplication CLAUDE.md.
- **CRLF still bites on WSL**: Both init scripts had CRLF on first write. Always verify with `file` command.
- **Scaffold needs separate source copies**: Can't use project root Rules.md/CLAUDE.md as scaffold sources — they contain project-specific content. Separate lib/templates/ copies are the clean solution.
