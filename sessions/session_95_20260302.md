## Technical Decisions

### Sentence case for all headings
- **Decision**: Use sentence case ("How it works") not title case ("How It Works")
- **Rationale**: Matches the user's blog writing style. Humanizer flagged title case as AI pattern #16.
- **Impact**: All headings and nav bar links use sentence case

### Em dash replacement pattern
- **Decision**: Replace `**Bold — text**` with `**Bold.** Text` throughout
- **Rationale**: 10+ em dashes in one README triggered humanizer pattern #13. Colons and periods are cleaner.
- **Impact**: Layer descriptions, non-coding workflow descriptions, conventions rules, and mode descriptions all use the colon/period pattern

### Shields.io star badge over GitHub buttons.js
- **Decision**: Use shields.io badge for star button, not GitHub buttons.js widget
- **Rationale**: GitHub strips `<script>` tags from README markdown for security. Shields.io badge renders natively. Will show "repo not found" until repo goes public, then self-heals.
- **Impact**: No code change needed when repo goes public

### Plugin update command researched
- **Decision**: Use `claude plugin update the-bulwark@qball-inc` as primary update path
- **Rationale**: Confirmed via claude-code-guide agent. Auto-update disabled by default for third-party marketplaces. Same command works for npm-installed plugins.
- **Impact**: FAQ entry covers both marketplace and npm update paths

## Learnings

- **GitHub strips script tags from README markdown**: The `buttons.github.io/buttons.js` widget won't render. Use shields.io badges for star buttons instead.
- **Shields.io badges require public repo**: Star count badge shows "repo not found" for private repos. Self-heals when repo goes public. No code change needed.
- **Plugin update command exists**: `claude plugin update <plugin>` is the correct way. Auto-update is per-marketplace, disabled by default for third-party marketplaces. Reinstalling via `claude /plugin install` also works but is not the canonical update path.
- **Humanizer is cleanest on factual/reference content**: Registry tables and FAQ entries passed with zero findings. Prose sections (what/why, how it works) are where AI patterns concentrate.
