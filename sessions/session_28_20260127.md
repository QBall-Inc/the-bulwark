## Technical Decisions

### Pipeline-templates Skill Loading
- **Decision**: Updated hook to load pipeline-templates skill before pipeline execution

- **Decision**: Kept original concise hook language, reverted "CRITICAL: Do NOT consolidate" additions
- **Rationale**: Verbose instructions caused regression (Test 2 - no pipelines at all)
- **Impact**: Simpler instructions + skill loading works better than complex instructions alone

### LLM Stochasticity Accepted
- **Decision**: Accept that same input can produce different pipeline execution patterns
- **Rationale**: Tests 1 and 3 had identical conditions but different outcomes
- **Impact**: Focus on providing good context (skills) rather than deterministic instructions

## Learnings

### Hook Skill Loading is Critical
- Hook instructions alone insufficient - LLM needs skill definitions in context
- `pipeline-templates` skill provides pipeline stage definitions
- Loading skill before execution improves compliance

### Simpler Instructions Work Better
- Adding verbose "CRITICAL" warnings caused regression
- Concise instructions + loaded skills > complex instructions alone

### E2E Testing Reveals LLM Variability
- Same conditions can produce different outcomes (stochastic behavior)
- Multiple test rounds needed to validate robustness
- Skill content acts as guardrails even when execution varies
