- **Decision**: Marketplace name `qball-inc` (not `plugins-market`)

## Learnings

- **WSL npm login needs --auth-type=legacy**: Browser-based npm login fails on WSL due to BROWSER env var issues with spaces in Windows paths. `npm login --auth-type=legacy` works reliably with terminal-based username/password prompt.
- **npm pack --dry-run is the .npmignore validator**: Don't trust .npmignore patterns in isolation. Always verify with dry-run — first pass had 18 extra files leaking through (empty .idea/.vscode, test fixtures, dev scripts, stale git artifacts).
- **.npmignore needs both file and dir patterns**: `.idea/` (with trailing slash) only matches directories. On some systems these can appear as 0B files. Add both `.idea` and `.idea/` to be safe.
- **Marketplace name vs repo name are separate concepts**: The `name` field in marketplace.json is the install-time identifier (after @). The repo path is only used once when adding the marketplace. Choose the name for UX, the repo name for clarity.
