# Session 98: Plugin Installation Bugfixes

## Summary

Ad-hoc session to troubleshoot and fix issues encountered during first real installation of The Bulwark plugin into a new project (CLEAR).

## Issues Encountered & Fixes

### Issue 1: Marketplace JSON parse error

**Symptom:** Adding the QBall-Inc marketplace failed with "Bad control character in string literal in JSON at position 560."

**Root cause:** The `description` field in `QBall-Inc/plugins-market/.claude-plugin/marketplace.json` contained raw newlines inside a JSON string literal. JSON spec forbids control characters (including newlines) in string values.

**Fix:** Replaced multi-line description with a single-line string. Committed directly on GitHub.

**Lesson:** JSON string values cannot contain literal newlines. GitHub's syntax highlighting showed the field in red (invalid) vs blue (valid) — a useful visual signal.

### Issue 2: Plugin clone fails with SSH permission denied

**Symptom:** After marketplace was added, installing the-bulwark plugin failed with `git@github.com: Permission denied (publickey)`.

**Root cause:** Claude Code's plugin installer clones repos via SSH (`git@github.com:`), but the user had no SSH keys configured for GitHub. The repo is public, so HTTPS would work.

**Fix:** `git config --global url."https://github.com/".insteadOf "git@github.com:"` — redirects all GitHub SSH URLs to HTTPS.

**Lesson:** This is a common issue for users who authenticate with GitHub via HTTPS/token but don't have SSH keys. Worth documenting as a prerequisite.

### Issue 3: Duplicate hooks file after install

**Symptom:** `/doctor` reported "Duplicate hooks file detected" — `./hooks/hooks.json` resolves to the already-loaded plugin hooks file.

**Root cause:** Plugin hooks are auto-loaded. If the plugin's hooks path matches a standard hooks path, Claude Code detects a duplicate.

**Fix:** Resolved by restarting Claude Code. The duplicate detection cleared on fresh session start.

**Lesson:** Plugin installation may require a restart for hooks to initialize cleanly. Already documented in README.
