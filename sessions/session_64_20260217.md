## Technical Decisions

### Adopt and Adapt Anthropic's Skill-Creator
- **Decision**: Fork Anthropic's official skill-creator as foundation, add Bulwark conventions (SA1-SA6, multi-agent patterns, anthropic-validator, content guidance).
- **Rationale**: Official skill-creator provides solid 6-step process and degrees-of-freedom framework. Building from scratch duplicates work.
- **Impact**: Need to track upstream changes. Continuous-feedback skill (P5.3) can catch drift.

### Adaptive Interview Depth
- **Decision**: 5 core questions for all skills, 3-5 follow-up questions triggered by complexity signals. Simple skills: 5 questions. Complex skills: 8-12.
- **Rationale**: Research finding: "the interview IS the classification." Fixed-depth interviews either under-serve complex skills or over-burden simple ones.
- **Impact**: Interview logic is the core of the meta-skill's decision-making.

### 5 Skill-Type Templates (Full Taxonomy)
- **Decision**: Simple, Reference-heavy, Pipeline, Script-driven, Research/Brainstorm.
- **Rationale**: DSL narrowness lesson from Prior Art: specialized templates succeed over general-purpose ones. Covers full Bulwark corpus.
- **Impact**: 5 template reference files to author per meta-skill. Most significant authoring effort.

### Structure + Content Guidance
- **Decision**: Include content scaffolding (good/bad instruction examples, description patterns, pitfalls) beyond just structural generation.
- **Rationale**: Contrarian finding: structure-only automation is low-value. The expensive part of skill creation is content quality, not file layout.
- **Impact**: `references/content-guidance.md` added to both skills.

### Each Skill Has Own References
- **Decision**: Shared decision-framework content duplicated in each skill's `references/` directory rather than cross-referenced.
- **Rationale**: Anthropic conventions require reference files within the skill folder. Cross-skill dependencies add complexity.
- **Impact**: Minor duplication between skill-creator and agent-creator references.

### Post-Generation Summary (Not Inline Comments)
- **Decision**: Present architectural decisions to user after generation. Generated files stay clean.
- **Rationale**: User preference. Keeps generated output concise and readable.
- **Impact**: Stage 5 in pipeline presents decision rationale.

### System-Prompt vs. Task-Instruction Register
- **Decision**: Agent-creator generates in system-prompt register (WHO the agent IS). Skill-creator generates in task-instruction register (WHAT to DO).
- **Rationale**: Official docs confirm agents and skills with context:fork have inverted content semantics. This is the most critical architectural distinction from the research.
- **Impact**: Agent-creator must enforce this register difference in generated output.

## Learnings

- **Research-to-task-brief pipeline works efficiently**: The `/bulwark-research` synthesis + post-synthesis user decisions provide enough structured information to write a comprehensive task brief directly, without needing a separate brainstorm phase. The 5-viewpoint research covers the design space that brainstorm would explore.
- **Critical Evaluation Gate adds minimal overhead when responses are Preference**: All 6 user responses across 3 rounds were classified as Preference (no Speculative). The gate classification step takes seconds and confirms no follow-up research is needed. The value is in the rare case where a Speculative claim would otherwise be blindly incorporated.
- **Contrarian viewpoint shaped multiple design decisions**: The quality ceiling finding drove the "structure + content guidance" scope decision. The maintenance coupling finding drove the "each skill has own references" decision. The decision tree fragility finding drove the "conversational elicitation, not inference" approach. Even when the Contrarian's overall verdict ("is a meta-skill necessary?") is overridden, the specific failure modes directly improve the design.
