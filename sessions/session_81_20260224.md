## Technical Decisions

### Extract AT Teammate Prompt Structure to reference file
- **Decision**: Moved ~50 lines of AT-specific prompt structure, config, and failure recovery from SKILL.md to `references/at-teammate-prompts.md`
- **Rationale**: AT content is only needed in --exploratory mode. Loading 100+ lines into context for --scoped users wastes tokens. Anthropic-validator flagged SKILL.md at 608 lines (over 500-line recommendation). Extraction brought it to 563.
- **Impact**: Stage 3B now loads reference file on demand. --scoped mode never sees AT prompt content.

### $ARGUMENTS placeholder added
- **Decision**: Added explicit `$ARGUMENTS` near top of SKILL.md body
- **Rationale**: Anthropic-validator medium finding — without explicit placement, arguments are silently appended at end of skill body. Explicit placement gives controlled argument positioning.
- **Impact**: Arguments now appear right after the intro paragraph, before "When to Use" section.

### Problem Validation added to Critic (both modes)
- **Decision**: Added "Problem Validation" as first focus area for Critical Analyst — "Should this problem be solved at all?"
- **Rationale**: FR5 from task brief. Distinct from assumption challenges (challenges the topic itself, not proposals about it). Low effort, applies to all brainstorm runs.
- **Impact**: Critic output template and synthesis template both updated with Problem Validation sections.

### Test strategy: same topic, both modes, PM-Essentials
- **Decision**: TC1 (--scoped) and TC2 (--exploratory) use identical topic in PM-Essentials project
- **Rationale**: Enables direct side-by-side comparison (TC4). PM-Essentials has no governance hooks, isolating skill behavior. Same topic removes topic variance as a confounding factor.
- **Impact**: TC4 comparison can definitively show whether --exploratory produces structurally different analysis.

## Learnings

- **AT content extraction reduces SKILL.md by ~50 lines**: For dual-mode skills, mode-specific prompt templates belong in references/ loaded on demand. Only mode detection logic and stage routing belong inline.
- **$ARGUMENTS placement matters**: Without explicit placement, Claude Code appends arguments after the entire skill body. For skills with 500+ lines, this means arguments are buried far from the parsing logic. Explicit `$ARGUMENTS` near the top is a best practice.
- **Anthropic-validator workflow is fast for incremental changes**: Single-skill validation (fetch + review) completed in ~2 minutes. Worth running on every skill modification, not just at the end.
