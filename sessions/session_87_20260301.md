## Technical Decisions

### Plugin install is user-guided, not Claude-executed
- **Decision**: Stage 5 presents instructions and waits for user confirmation. Claude cannot run `claude plugin install/enable` from within a session.
- **Rationale**: CLI commands start new processes; plugin management must be done via `/plugin` TUI or from a separate terminal.
- **Impact**: Cleaner separation — Claude handles binaries + settings, user handles plugin marketplace.

### Non-blocking AskUserQuestion for in-session install
- **Decision**: "I'll install in this session" option dismisses AskUserQuestion, prints instructions, waits for natural text confirmation.
- **Rationale**: AskUserQuestion blocks the session — user can't type `/plugin` while it's waiting.
- **Impact**: Better UX for users who want to install plugins without leaving the session.

### Scope guidance tied to /plugin install screen
- **Decision**: Stage 5 tells user exactly which scope option to select on the plugin install screen, matching their earlier choice.
- **Rationale**: `/plugin` defaults to user scope. Without explicit guidance, users may install to wrong scope.
- **Impact**: Consistent scope across ENABLE_LSP_TOOL settings and plugin installation.

### .lsp.json not needed for Bulwark plugin
- **Decision**: Bulwark provides a skill that guides users to install official LSP plugins. No `.lsp.json` in our plugin.
- **Rationale**: We don't bundle language servers — official plugins (`typescript-lsp`, `rust-analyzer-lsp` etc.) each have their own `.lsp.json`.
- **Impact**: Simpler plugin structure. Our value is orchestration, not server distribution.

## Learnings

- **Claude cannot run `claude plugin` CLI commands from within a session**: Plugin management requires user action via `/plugin` TUI or separate terminal. Skills that need plugin installation must guide the user, not attempt it directly.
- **AskUserQuestion blocks `/` commands**: When skill needs user to run `/plugin` or other TUI commands, dismiss AskUserQuestion first. Use natural text confirmation instead.
- **Official LSP plugins use `-lsp` suffix**: 11 plugins in `claude-plugins-official` marketplace. Plugin names differ from binary names (e.g., `typescript-lsp` not `typescript-language-server`).
- **enabledPlugins uses object format with marketplace qualifier**: `{"typescript-lsp@claude-plugins-official": true}`, not array of strings.
- **Frontloading AC with file paths saves brainstorm time**: Adding specific paths and design questions to ACs gives SME agent concrete starting points.
