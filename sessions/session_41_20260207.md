- **Decision**: Remove `hooks:` from implementer frontmatter. Rely on global PostToolUse hook (enforce-quality.sh) + agent self-validation protocol

- **Impact**: P4.4 agent design simplified (no hooks in frontmatter). Quality enforcement still works via global hooks. Agent protocol adds explicit `just typecheck && just lint` self-validation step as belt-and-suspenders

- **Impact**: Every custom sub-agent's log output is verified at completion. General-purpose agents skipped (no naming convention).

- **Impact**: Zero changes to existing enforcement infrastructure

## Learnings

- **Global PostToolUse hooks fire for sub-agent writes**: This was uncertain based on prior (incorrect) memory entries. Empirically verified via hooks.log timestamp correlation showing enforce-quality.sh triggering between SubagentStart and SubagentStop for files clearly written by the sub-agent.
- **Agent-scoped hooks in frontmatter are broken**: Despite official docs showing the syntax, hooks declared in `.claude/agents/*.md` YAML frontmatter do not fire when the agent is spawned via Task tool. Known bug #18392/#19213.
- **Breadcrumb debugging technique**: Writing to a guaranteed-visible path like `/tmp/claude/` is more reliable for hook debugging than relying on project-relative paths that may not resolve correctly.
- **PostToolUse matcher only matches tool names**: Cannot filter by file path or type in the matcher field. All path-based filtering must happen inside the hook script itself.
