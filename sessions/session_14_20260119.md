## Learnings

### Agent vs Skill Invocation (Key Learning)
- **`user-invocable: true`** only works for skills (SKILL.md files)
- **Custom sub-agents** (markdown in agents/) are ALWAYS invoked via Task tool
- **`$ARGUMENTS`** is skill-only; agents receive input via CONTEXT in prompt
- Documentation updated across task briefs, agents, and test protocols

### Reviewer Self-Limitation Bug
- bulwark-standards-reviewer flagged Bash as invalid because it couldn't use Bash itself
- Fixed by adding constraint: "validate against DOCUMENTATION, not by attempting to use"
- Lesson: Sub-agents may conflate "I can't access X" with "X is invalid"

### Improvised Fix Validator Patterns
- Generated JS scripts for test execution when native runners unavailable
- Produced both YAML (machine-readable) and TXT (human-readable) outputs
- Incorporated into P1.3 task brief as recommended patterns
