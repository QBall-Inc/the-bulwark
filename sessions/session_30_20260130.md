## Technical Decisions

### Review Depth Scaling
- **Decision**: Default comprehensive; `--quick` flag for tiered by lines (<50=security, 50-500=security+type-safety, >500=all)
- **Rationale**: Pipelines should be thorough quality gates; users can opt for speed explicitly
- **Impact**: All pipeline invocations run full review; user-invocable has speed option

### Framework Detection
- **Decision**: Auto-detect from package.json with fallback to generic; `--framework` override available
- **Rationale**: Pipelines run automatically and need framework context without intervention
- **Impact**: Security section loads framework-specific patterns (React XSS vs Express injection)

### Confidence Scores
- **Decision**: 2-tier model (Verified vs Suspected) with evidence field
- **Rationale**: Simple to implement; forces skill to either prove issue or acknowledge uncertainty
- **Impact**: Findings include confidence level and evidence trace

### Git Context
- **Decision**: Optional and targeted; only for complexity findings; `--include-git-context` flag
- **Rationale**: Most findings don't benefit from history; valuable for "why is this complex?" questions
- **Impact**: Linting section can show commit history for flagged complex code

### Skill File Organization
- **Decision**: Templates, examples, framework patterns in subfolders, not embedded in SKILL.md
- **Rationale**: Consistent with Bulwark skill pattern; enables independent updates
- **Impact**: Skill has references/, examples/, frameworks/, templates/ subdirectories

### Statusline Fix
- **Decision**: Use empty hook arrays instead of `disableAllHooks: true`
- **Rationale**: Schema shows `disableAllHooks` disables both hooks AND statusLine
- **Impact**: Hooks disabled during development, statusLine remains functional

### P4.2 External List References
- **Decision**: Reference URLs to external lists, do not embed copies
- **Rationale**: External lists (BLNS, OWASP) are maintained by their communities
- **Impact**: Data files reference sources, `references/external-lists.md` contains URLs

### P4.2 Language-Specific Variants
- **Decision**: Include language-specific edge cases as optional extensions
- **Rationale**: JS `==` vs `===`, Python `None` vs `False` are valuable but not universal
- **Impact**: `data/language-specific/` directory with javascript.yaml, python.yaml, rust.yaml

### P4.2 Concurrency Patterns
- **Decision**: Include scenario patterns for state machine and race condition testing
- **Rationale**: Concurrency bugs are overlooked and costlier to fix later
- **Impact**: `data/concurrency/` directory with race-conditions.yaml, state-machines.yaml

### P4.2 Update Mechanism
- **Decision**: Version metadata + plugin releases + optional helper skill (P6.x)
- **Rationale**: Core skill simple, helper skill enables checking for new patterns
- **Impact**: Data files have version/last_updated/source_urls metadata
