## Learnings

### Confirmation Bias in Test Fixtures
Test fixtures with explanatory comments about expected violations can cause sub-agents to "find" what they're told to find rather than independently detecting issues. Solution: keep test files clean, store expected results separately.

### Skill Dependency Model
Skills should declare dependencies in frontmatter `skills:` array, not reference paths in instructions. When installed, skills are accessed by name through the dependency system, not by file path.
