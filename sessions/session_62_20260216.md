## Technical Decisions

### Ralph Loops Descoped from Bulwark
- **Decision**: Ralph Loops formally removed from P5.3-5 scope and all task descriptions
- **Rationale**: Two research sessions (61-62) conclusively established near-zero applicability. Only P5.4/P5.5 structural validation had fit, but that's basic iterative dev, not a methodology worth adopting. Bulwark skills are short-lived, user-supervised, orchestrator-driven — no iterative autonomous implementation use case.
- **Impact**: P5.4/P5.5 names updated (removed "Ralph loop" references). Acceptance criteria revised. Research schedule compressed. `descoped_patterns` section added to tasks.yaml.

### Agent Teams Scoped to P5.13 + P5.15 Only
- **Decision**: Agent Teams adoption limited to two use cases: P5.13 plan-creation (dual-mode) and P5.15 brainstorm `--exploratory` mode. All existing skills remain on Task tool.
- **Rationale**: All 5 viewpoints converged — existing Bulwark pipelines are "focused workers that report back" (Task tool's sweet spot). Agent Teams add value only for peer debate (competing hypotheses, collaborative idea validation). Two concrete use cases identified.
- **Impact**: P5.13 acceptance criteria updated for dual-mode. New P5.15 task created. Both gated on Agent Teams experimental flag.

### Brainstorm --scoped / --exploratory Modes
- **Decision**: bulwark-brainstorm to support two modes: `--scoped` (current Task tool sequential) and `--exploratory` (new Agent Teams collaborative)
- **Rationale**: User identified gap — current sequential design excels at focused assessment of well-defined topics but doesn't serve collaborative idea validation where the problem framing itself is uncertain. Agent Teams peer debate is the right mechanism for exploratory mode.
- **Impact**: New P5.15 task. Names describe intent (scoped vs exploratory) not mechanism (sequential vs collaborative).

### Research Schedule Compressed
- **Decision**: 8 planned sessions → 4 sessions (3 brainstorm + 1 research). Cross-topic synthesis dropped.
- **Rationale**: Ralph descoped (no Ralph brainstorm needed). No overlap between Agent Teams and Feedback Patterns findings (no cross-synthesis needed). P5.3 needs brainstorm not research (implementation questions only). P5.4/P5.5 needs research (meta-skill design is novel territory).
- **Impact**: Sessions 63-66 finalized. Significant time savings.

### P5.3 Scope: General-Purpose
- **Decision**: P5.3 (continuous-feedback) is general-purpose, applicable to any codebase, not Bulwark-specific
- **Rationale**: User preference. The Collect→Analyze→Act→Validate pipeline applies to any project with session handoffs or accumulated learnings.
- **Impact**: Acceptance criteria updated. Per-skill specialization (code-review security patterns, test-audit AST updates, bug-magnet data) included as examples, not hardcoded targets.

## Learnings

- **Research can conclusively descope**: Two sessions of structured multi-viewpoint research produced enough evidence to formally descope Ralph Loops with confidence. The sessions weren't wasted — ruling out with evidence is more defensible than ruling out by assumption.
- **Post-synthesis discussion surfaces insights research misses**: The brainstorm `--exploratory` mode was identified during post-synthesis user discussion, not by any of the 5 research agents. The agents correctly analyzed the *current* brainstorm design but didn't consider alternative modes. User domain knowledge + synthesis context = emergent insights.
- **Intent-based naming > mechanism-based naming**: `--scoped`/`--exploratory` describes what the user wants to achieve. `--sequential`/`--collaborative` describes how it works internally. The former is better UX design.
- **Schedule compression from evidence**: Original 8-session plan compressed to 4 because research provided clear answers. Research investments that reduce downstream work are net positive even when findings are "don't do X."
- **bulwark-research pipeline continues to perform well**: All 5 agents succeeded on first attempt for second consecutive session. Research-Evaluate-Deepen producing substantive multi-pass findings. Critical Evaluation Gate correctly classified all user input.
