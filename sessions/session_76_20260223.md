## Technical Decisions

### Sub-agents over role reference files
- **Decision**: Create dedicated `.claude/agents/plan-creation-*.md` files instead of `references/role-*.md` files
- **Rationale**: Sub-agents are reusable across both Task tool and Agent Teams modes, independently invocable, and follow the product-ideation pattern that already works. Role reference files are injected into prompts at runtime — less deterministic.
- **Impact**: Each agent stands alone for independent use (e.g., `plan-creation-architect` for any architectural analysis). Sets a new pattern for future pipeline skills.

### Model selection: PO=opus, Architect=opus, Eng Lead=sonnet, QA/Critic=sonnet
- **Decision**: Opus for roles requiring deep reasoning (codebase exploration, system design), Sonnet 4.6 for structured/analytical roles (scheduling, adversarial review)
- **Rationale**: User decision — Architect "has to earn their chops" with deep system design reasoning. Eng Lead is more mechanical (sequencing, estimation). QA/Critic benefits from comprehensive input more than model depth.
- **Impact**: 2 Opus + 2 Sonnet agents per pipeline invocation. Token cost balanced.

### Do not use create-skill pipeline to create this skill
- **Decision**: Hand-author using create-skill's templates and patterns as references, but don't run the interactive pipeline
- **Rationale**: Task brief already has all decisions made (349 lines). Running the interactive interview/classification stages would burn 15-20K tokens for no new information.
- **Impact**: Faster delivery, same quality.

### Individual agent output formats are intermediate — CLEAR compatibility only in final output
- **Decision**: Agent outputs are optimized for orchestrator consumption, not CLEAR compatibility
- **Rationale**: CLEAR only consumes the synthesis output (plan-output.md template). Agent logs are pipeline-internal artifacts in `logs/`. The synthesis stage (T-008) handles the CLEAR-compatible YAML body.
- **Impact**: T-007 (plan-output.md) and T-008 (synthesis stage) are where CLEAR compatibility work lives.

## Learnings

- **Sub-agents as standalone capabilities is a better pattern**: Instead of role reference files that only work when the orchestrator injects them into prompts, dedicated agents work independently AND in pipelines. The product-ideation skill already proved this pattern; plan-creation now adopts it formally. Future pipeline skills should follow this approach.
- **Parallel sub-agent authoring is efficient**: Spawning 3 agent-authoring sub-agents simultaneously (T-002/T-003/T-004) completed all 3 in ~20 minutes of wall time vs ~60 minutes sequential. QA/Critic had to wait for all 3 (needs awareness of what it critiques) — correct dependency modeling.
- **Token budgeting for agent authoring**: Each Sonnet sub-agent used ~55-70K tokens to author one agent file + read references. With 4 agents, this consumed roughly 25-30% of the parent context budget. The Task tool approach kept this cost external to the main conversation context.
