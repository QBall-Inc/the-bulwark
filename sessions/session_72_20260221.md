## Technical Decisions

### Self-contained agent references in create-skill (Option C: Hybrid → replicate)
- **Decision**: Replicate agent-template.md and agent-conventions.md into create-skill's own references/ instead of depending on create-subagent skill or invoking it.
- **Rationale**: Standalone repo users may pick create-skill without create-subagent. Skill dependency would create broken references. Full invocation wastes tokens on redundant interviews. Replication ensures self-containment.
- **Impact**: Two copies to maintain. continuous-feedback skill will flag drift.

### Pipeline sub-agent generation via convention reuse (not skill invocation)
- **Decision**: create-skill's Stage 2 Generator reads agent-template.md and agent-conventions.md as CONTEXT and produces both orchestrating skill + agent files in one pass. Does NOT invoke create-subagent as a skill.
- **Rationale**: create-subagent's interactive interview is redundant (create-skill already interviewed the user). Per-agent validation loops are inefficient (create-skill validates all assets together in Stage 3). Token cost of N full pipelines is prohibitive.
- **Impact**: Efficient single-pass generation. Batch validation. No redundant interviews.

### Routing logic placement in Stage 0 (not Stage 1)
- **Decision**: Pipeline/teams detection happens in Stage 0 (Pre-Flight) after interview, before classification. Hard redirect — pipeline stops immediately.
- **Rationale**: Detecting infeasible requests early avoids wasting tokens on classification, generation, and validation of assets that can't function. Q3 follow-ups (Q6/Q7) provide the routing signal.
- **Impact**: Zero wasted tokens on infeasible pipeline agents.
