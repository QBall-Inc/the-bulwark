## Technical Decisions

### Non-overlapping sed patterns for test path transforms
- **Decision**: Use patterns unique to each target line rather than ordering by specificity
- **Rationale**: Earlier approach failed because SCRIPT_PATH sed (`__dirname, '..'`) re-matched already-transformed FIXTURES_ROOT lines. Non-overlapping patterns (4-parent chain, `'node_modules'`, `.ts` filename) are deterministic regardless of execution order
- **Impact**: Sync script produces correct paths every run without cascading collisions

### Test layout: separate tests/ directory
- **Decision**: Tests live at `tests/test-audit/` in standalone repo, not inside `skills/test-audit/scripts/__tests__/`
- **Rationale**: Clean separation of concerns — skills are installable artifacts, tests are development infrastructure
- **Impact**: Test file path constants transformed during rsync via sed

### tasks.yaml archival strategy
- **Decision**: Full details archived to tasks_completed.yaml, one-liner YAML flow mappings as stubs in main file
- **Rationale**: 63% line reduction while preserving all information and keeping task IDs visible
- **Impact**: Future sessions load smaller tasks.yaml; full details accessible in archive
