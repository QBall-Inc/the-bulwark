## Technical Decisions

### Task Brief Split: P5.3 Standalone, P5.4/P5.5 Separate
- **Decision**: P5.3 gets its own task brief. P5.4/P5.5 bundled separately as `P5.4-5-skill-agent-creators.md` (to be created after Session 64 research).
- **Rationale**: Originally bundled because of anticipated design overlap. Overlap is structural (common Bulwark skill patterns), not design-specific. Each skill has distinct pipeline shape and domain concerns.
- **Impact**: P5.4/P5.5 implementation_plan references updated in tasks.yaml. P5.4/P5.5 task brief will be created after `/bulwark-research` in Session 64.

### Full Pipeline Directly (No v0 or Prompt Experiment)
- **Decision**: Build the full 4-agent pipeline from the start, skipping the Critic's recommended single-agent v0 and prompt experiment.
- **Rationale**: User trusts existing multi-stage skill patterns (bulwark-research, bulwark-brainstorm, test-audit). Adding a v0 build/test cycle doesn't justify the overhead.
- **Impact**: Implementation session starts directly with templates, not validation experiments.

### 3 Specializations in v1
- **Decision**: test-audit + code-review + general (PM's recommendation over Critic's 1+general)
- **Rationale**: Both skills have rich documented improvement histories and reference file structures.
- **Impact**: 3 specialization reference files to author instead of 2.

### Pass-Through Schema
- **Decision**: Collector preserves near-raw content. Analyzers handle interpretation.
- **Rationale**: Maximum fidelity. Addresses Critic's concern about lossy intermediate formats destroying actionability.
- **Impact**: Higher token consumption in Analyze stage, but avoids quality-destroying compression.

### Specialization References Sync to Standalone Repo
- **Decision**: Include specialize-*.md files in essential-agents-skills repo as examples.
- **Rationale**: Users can adapt them for their own skills. Demonstrates extensibility.
- **Impact**: sync-essential-skills.sh will include these files.

## Learnings

- **Critic challenges add real value even when overridden**: The Critical Analyst's "modify" verdict with v0 and prompt experiment was ultimately overridden by user preference, but the concerns it raised (lossy schema, keyword routing failures, proposal quality risk) directly shaped the final design. The pass-through schema and LLM-classified routing decisions came from addressing the Critic's challenges.
- **Task brief splits are better decided after brainstorm than before**: The original Session 46 decision to bundle P5.3-5 was based on anticipated overlap. After brainstorming P5.3, it became clear the overlap is structural, not design-specific. Brainstorm output provides the information needed to make this split decision with confidence.
- **Prior art references enhance sub-agent output**: The enhance_skill.py reference provided by the user was incorporated into all 5 agent prompts and directly influenced the Architect's source prioritization design and the PM's proposal format thinking. External references are high-value, low-cost inputs.
