## Technical Decisions

### Fixture Location Strategy
- **Decision**: Use `scripts/` for source, `tests/` for test files (project-level, not tests/fixtures/)
- **Rationale**: Dogfooding - use project infrastructure for testing, not isolated fixture dirs
- **Impact**: Fixtures use real tsconfig, Justfile, hooks

### Backup Folder for Fixtures
- **Decision**: Create `backup/` at project root with mirror structure
- **Rationale**: test-audit may trigger REWRITE_REQUIRED and modify test files
- **Impact**: Easy restoration: `cp backup/tests/handlers/* tests/handlers/`

### Audit-Only Prompt for P4.2-1
- **Decision**: Prompt explicitly says "don't make any changes to the file yet"
- **Rationale**: Preserve fixture for subsequent tests without triggering rewrites
- **Impact**: Can verify edge case detection without modifying fixtures
