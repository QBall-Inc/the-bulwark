## Technical Decisions

### Rsync transform coverage must include all skill files
- **Decision**: Expanded `CA_FILES` array from 3 reference files to all 7 create-agent files (SKILL.md + 6 references)
- **Rationale**: Initial transforms only targeted decision-framework, agent-conventions, and content-guidance. But template files also contained SA1-SA6 references, DEF-P4-005, and GitHub issue numbers.
- **Impact**: Caught 2 missed patterns on post-sync validation: `(agent-scoped hooks are broken per GitHub #18392/#19213)` and `"bulwark-{agent-name}"` in JSON example. Both fixed with additional sed rules.

### 3 agent templates (vs 5 skill templates)
- **Decision**: create-agent has 3 templates (single, pipeline, teams) vs create-skill's 5 (simple, reference-heavy, pipeline, script-driven, research)
- **Rationale**: Agents are always `.claude/agents/{name}.md` — a single file. The variance is in architecture (single vs pipeline vs teams), not in file structure. Fewer templates = less maintenance.
- **Impact**: Simpler classification tree (Decision A maps directly to template). No "supporting files" decision needed for agents.
