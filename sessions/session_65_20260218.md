## Technical Decisions

### 4 Roles Instead of 5
- **Decision**: Collapse Engineering Lead + Delivery Lead into single "Engineering & Delivery Lead" role. Final team: PO, Architect, Eng+Delivery Lead, QA/Critic.
- **Rationale**: Critic flagged 5 agents exceeds research-validated 3-4 coordination cap. Eng Lead and Delivery Lead had highest overlap risk. 4 roles stays within cap while preserving distinct functional coverage.
- **Impact**: Role reference files reduced from 5 to 4. Combined role covers sequencing, effort, scheduling, milestones, dependencies.

### Agent Teams as Enhanced/Opt-in (Not Primary)
- **Decision**: Task tool is the default mode. Agent Teams activated only when env var is set AND user explicitly opts in.
- **Rationale**: PM and Critic both argued Task tool is production-stable and serves 100% of users. Agent Teams is experimental with ~2x token cost. Framing affects resource allocation priority.
- **Impact**: Pre-flight offers Agent Teams as a choice rather than auto-selecting. Updates pre-brainstorm alignment language.

### Build Both Modes (Override Critic)
- **Decision**: Build both Task tool and Agent Teams modes across 3 sessions, despite Critic recommending comparative test first.
- **Rationale**: P5.13 is explicitly the Agent Teams proving ground. Building it is the empirical validation.
- **Impact**: 3-session estimate retained. Kill criteria from Critic serve as safety net.

### Proactive CLEAR Compatibility
- **Decision**: Design YAML body fields to match CLEAR's MasterPlan type interface proactively.
- **Rationale**: User owns both projects. Low-cost decision (field naming) with high future value (no adapter).
- **Impact**: YAML uses `phases[].id`, `phases[].status`, `milestones[].type`, `milestones[].requires` matching CLEAR types.

### Hybrid Plan Format (Markdown + YAML)
- **Decision**: Markdown preamble for executive summary + YAML body for structured plan.
- **Rationale**: YAML is 40-50% more token-efficient for structured data. Prose summaries are unreadable as nested YAML strings. Hybrid gives both.
- **Impact**: Plan output template uses fenced YAML within Markdown document.
