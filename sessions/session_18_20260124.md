## Technical Decisions

### Multi-line Confirmed Working
- **Decision**: Proceed with 3-line statusline design
- **Rationale**: Empirical testing confirmed Claude Code displays all lines from script output, despite docs suggesting single-line only
- **Impact**: Developer preset shows model+gauge, last file, and git info on separate lines

### Token Count from Percentage
- **Decision**: Calculate tokens from `(percentage / 100) * context_window_size`
- **Rationale**: `total_input_tokens`/`total_output_tokens` include sub-agent calls, not actual context usage
- **Impact**: Token display now matches `/context` command output

### Last File by Modification Time
- **Decision**: Sort modified + untracked files by mtime, show most recent
- **Rationale**: `git diff --name-only` returns arbitrary order, not most recently modified
- **Impact**: Statusline shows the file you actually just worked on

### RGB Escape Codes for Exact Colors
- **Decision**: Use `\033[38;2;R;G;Bm` format instead of ANSI 256
- **Rationale**: Allows exact hex color matching per user specification
- **Impact**: Colors match user's pastel palette exactly

## Learnings

### Multi-line Statusline Works
- Official docs may be conservative/outdated
- Empirical testing revealed multi-line support
- Script simply outputs multiple lines, Claude Code displays all

### Token Fields Are Cumulative
- `total_input_tokens`/`total_output_tokens` = ALL API calls including sub-agents
- `used_percentage` = actual context window usage
- Calculate from percentage for accurate display

### WSL Line Endings
- CRLF breaks shebang lines on WSL
- Always run `sed -i 's/\r$//'` on new shell scripts
- Or use Write tool which handles this correctly
