## Technical Decisions

### Hook Output Format - JSON with Blocking
- **Decision**: Changed PostToolUse hook output from plain text to JSON with `decision: "block"` and `hookSpecificOutput.additionalContext`
- **Rationale**: Plain text stdout from PostToolUse goes to verbose mode only, NOT Claude's context. JSON with `additionalContext` is the only way to inject instructions into Claude's context.
- **Impact**: Hooks now reliably inject pipeline instructions. Claude cannot ignore them.

### Quality Check Status Communication
- **Decision**: enforce-quality.sh exports QUALITY_CHECKS_PASSED env var, suggest-pipeline.sh includes status in output
- **Rationale**: Claude was re-running quality checks because it didn't see they already passed
- **Impact**: Claude sees "Quality checks COMPLETED" message and doesn't re-run typecheck/lint/build

### Removed once:true from SessionStart
- **Decision**: Governance protocol re-injects after /clear and /compact
- **Rationale**: User wanted protocol to persist across conversation clears
- **Impact**: Banner appears after every /clear, ensuring governance is always visible

## Learnings

### PostToolUse Hook Output Injection
- Plain text stdout goes to verbose mode only, NOT Claude's context
- Only JSON with `hookSpecificOutput.additionalContext` injects into context
- Use `decision: "block"` to force Claude to acknowledge before proceeding
- Exit code 2 blocks the operation, exit code 0 allows it

### Quality Checks Are Project-Wide
- `just typecheck` runs on entire project, not just edited file
- One broken file blocks ALL subsequent edits
- Clean up test artifacts immediately after testing

### Pipeline Selection Should Consider Work Type
- File type alone is insufficient for pipeline selection
- Write (new file) → New Feature Pipeline (includes tests)
- Edit (existing file) → Code Review Pipeline
- Current implementation only checks file extension
