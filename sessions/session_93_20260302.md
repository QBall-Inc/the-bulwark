## Technical Decisions

### Plugin.json: agents and skills fields removed
- **Decision**: Removed `"agents": "./agents/"` and `"skills": "./custom/skills/"` from plugin.json
- **Rationale**: Both directories are auto-discovered from default locations at plugin root. The `agents` field requires individual file paths (not directory), and explicit `skills` path prevented discovery. Empirically validated: removing both fields → all agents and skills discoverable at runtime.
- **Impact**: Simpler plugin.json. No need to enumerate agents individually.

### Init skill: multi-stage guided onboarding
- **Decision**: Expanded init from thin wrapper (~140 lines) to full onboarding orchestrator (348 lines) with interactive setup selection, language detection, and downstream skill invocation.
- **Rationale**: Downstream skills (LSP, scaffold) require language toolchain to be present. Collecting language selection once and passing to downstream skills is cleaner than each skill re-asking. Statusline has no dependencies so runs first.
- **Impact**: Users get a complete guided setup in one command. State file enables --verify after restart.

### Skill invocations use plugin namespace prefix
- **Decision**: All downstream skill invocations use `/the-bulwark:` prefix (e.g., `/the-bulwark:bulwark-statusline`)
- **Rationale**: Bare names like `/bulwark-statusline` may fail when skills are scoped under the plugin namespace. Validator flagged as High severity.
- **Impact**: Ensures correct routing when running as a plugin.

### State file path anchored to $CLAUDE_PROJECT_DIR
- **Decision**: State file written to `$CLAUDE_PROJECT_DIR/tmp/init/init-state.yaml` instead of relative `tmp/init/`
- **Rationale**: Relative paths break if working directory changes between init and verify runs.
- **Impact**: Reliable init-to-verify handoff regardless of cwd.

## Learnings

- **Plugin auto-discovery trumps explicit paths**: Both `agents/` and `skills/` at plugin root are auto-discovered. Explicit path fields in plugin.json are for non-default/custom locations only. The `agents` field requires individual file paths (not directory glob), while `skills` accepts directory paths — but neither is needed for default locations.
- **Guided onboarding is a differentiator**: The init skill orchestrating statusline → language selection → scaffold → LSP in one flow creates a polished first-run experience. Worth the extra complexity over a simple wrapper.
- **Cross-scope pre-flight prevents silent conflicts**: Checking both project and user scope regardless of selection catches cases where governance files at one scope conflict with the other.
- **Scaffold skill has accumulated tech debt**: Three bugs found during plugin validation (timeouts, duplicate hooks, missing just check). Plugin-as-consumer testing surfaces issues that unit testing misses.
