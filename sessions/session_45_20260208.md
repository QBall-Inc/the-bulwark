## Technical Decisions

### TestAudit Gap Fix (Option 1: Widen Stage 3b/4b condition)
- **Decision**: TestAudit runs if any test files were created/modified by FixWriter/Implementer OR TestWriter, not just TestWriter
- **Rationale**: When implementer writes tests (which it always does), TestWriter skips, causing TestAudit to skip. Implementer-written tests need T1-T4 audit too.
- **Alternatives rejected**: (2) SA6 mandatory for test audit — creates special-case logic; (3) Accept gap — "unlikely" T1 violation is not "impossible"; (4) Revert to orchestrator FixWriter — loses quality gates, structured output, context isolation
- **Impact**: fix-validation Stage 3b and new-feature Stage 4b both updated

### fix-bug Skill Stage 2 and Stage 5 Alignment
- **Decision**: Stage 2 uses bulwark-implementer (not orchestrator), Stage 5 uses general-purpose Sonnet (not full code-review pipeline)

- **Impact**: fix-bug skill and fix-validation pipeline now fully aligned

## Learnings

- **CRLF vigilance on WSL**: New scripts on /mnt/c/ can get CRLF. Manual testing catches this. Production scripts were clean because they'd been tested before.
- **Pipeline suggestion fabrication**: When a script fails (e.g., CRLF), agents may fabricate output by reading the script's logic. Fabrication can be factually accurate but isn't from actual execution.
- **TestAudit gap pattern**: When a code-writing agent also writes tests, the separate TestWriter stage skips, which cascades to TestAudit skipping. Any pipeline with Implementer → TestWriter → TestAudit needs the widened condition.
- **3 compactions in one session**: Heavy testing sessions with log verification consume context rapidly. Plan for handoff at 73% to leave room for cleanup and commit.
