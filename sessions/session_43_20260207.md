## Learnings

- **claude-code-guide inconsistency**: The built-in agent works reliably for some doc URLs (sub-agents) but fails for others (skills), reading local project files instead. WebFetch fallback with URL redirect handling (docs.anthropic.com → code.claude.com) is reliable. Detection heuristic: look for phrases like "Your project confirms" in agent output.
- **`allowed-tools` vs `tools`**: Official docs use `allowed-tools` for skills but `tools` for agents. Only one Bulwark skill (`bulwark-statusline`) is affected. Low risk but worth standardizing post-development.
